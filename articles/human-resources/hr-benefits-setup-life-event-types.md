# <a name="configure-life-event-types"></a>생활 이벤트 유형 구성


[!INCLUDE [PEAP](../includes/peap-2.md)]

[!include [Applies to Human Resources](../includes/applies-to-hr.md)]

Dynamics 365 Human Resources는 **생활 이벤트 유형** 을 사용하여 결혼 또는 자녀 출산과 같이 직원 복리후생 등록을 업데이트할 수 있는 이벤트를 정의합니다. 각 생활 이벤트 유형 ID는 하나의 생활 이벤트 유형에만 연관될 수 있습니다. 예를 들어 **직원 주소 변경** 생활 이벤트 유형과 연결된 **주소 변경** 이라는 **생활 이벤트 ID** 를 만들면 **직원 주소 변경** 이라는 레이블이 지정된 다른 ID를 만들고 이를 **직원 주소 변경** 생활 이벤트 유형과 연결할 수 없습니다. 생활 이벤트 유형이 계획 유형과 연결되지 않은 경우 생활 이벤트 유형은 생활 이벤트를 트리거하지 않습니다. 자세한 내용은 [계획 유형 만들기](hr-benefits-setup-plan-types.md)를 참조하세요.

## <a name="create-a-life-event-type"></a>생활 이벤트 유형 만들기

1. **복리후생 관리** 작업 영역의 **설정** 에서 **생활 이벤트 유형** 을 선택합니다.

2. **새로 만들기** 를 선택합니다.

3. 다음 필드에 값을 지정합니다.

   | 필드 | 설명 |
   | --- | --- |
   | **생활 이벤트 유형 ID** | 생활 이벤트 유형 고유 식별자. |
   | **설명** | 생활 이벤트 유형에 대한 설명. |
   | **생활 이벤트 유형** | 직원의 복리후생 등록을 업데이트하는 촉진제. 생활 이벤트의 목록은 아래의 [생활 이벤트](hr-benefits-setup-payment-frequencies.md?life-events)를 참조하세요. |

4. **저장** 을 선택합니다.

## <a name="view-attached-plans"></a>연결된 계획 보기

선택한 **생활 이벤트 유형** 에 연결된 계획 목록을 볼 수 있습니다. 생활 이벤트는 계획 유형과 연결되고 계획 유형은 계획과 연결됩니다.

1. **복리후생 관리** 작업 영역의 **설정** 에서 **생활 이벤트 유형** 을 선택합니다.

2. **작업** 을 선택합니다.

3. **연결된 계획** 을 선택합니다.

## <a name="life-events"></a>생활 이벤트

생활 이벤트 유형을 만들 때는 다음 생활 이벤트 중에서 선택할 수 있습니다.

| 수명 이벤트 | 위치 | 트리거 |
| --- | --- | --- |
| **결혼 상태 변경** | **작업자 > 프로필 > 개인 정보 > 결혼 상태**| 결혼 상태 변경 |
| **고용 상태 변경** |**근로자 > 고용 > 고용 이력 페이지** | 기존 고용 세부 정보가 있는 작업자의 경우 고용 상태가 다른 새 고용 세부 정보를 생성하면 생활 이벤트가 트리거됩니다.  기존 고용 세부 정보를 다른 고용 상태로 업데이트해도 생활 이벤트가 발생합니다.  |
| **직원 주소 변경** |**작업자 > 프로필 > 주소**</li><li>**작업자 > 개인 정보 > 개인 연락처 > 주소**</li></ul> | 주소 변경. 주소는 **기본** 이어야 생활 이벤트를 트리거합니다. |
| **피부양자 변경** |<br><ul><li>**작업자 > 프로필 > 개인 정보 > 개인 연락처**/li><li>**직원 셀프 서비스**</li></ul> | 피부양자로 지정하고 **유효 기간 시작** 을 정의하는 개인 연락처 추가. 개인 연락처의 피부양자 **유효 기간 시작** 정보 업데이트. 개인 연락처 관계는 자녀, 배우자, 동거인 또는 전 배우자여야 합니다.  |
| **출생 또는 입양(피부양자)** |<br><ul><li>**작업자 > 프로필 > 개인 정보 > 개인 연락처**</li><li>**직원 셀프 서비스 > 개인 정보 편집 > 개인 연락처**</li></ul>| **생일** 또는 **입양 날짜** 가 추가되거나 업데이트됨. 자녀의 **생일** 이 필요합니다. |
| **보장 상실(배우자/동거인)** | 근로자 > 프로필 > 개인 정보 > 개인 연락처 > 피부양자 세부 정보 > 보장 상실 | **적용 날짜** 와 함께 개인 연락처에 **보장 상실** 이 선택됨 |
| 동거인 고용 변경 | **근로자 > 프로필 > 개인 정보 > 개인 연락처 > 피부양자 세부 정보 > 고용됨** | 개인 연락처를 만들고 **고용됨** 을 **예** 로 설정. 개인 연락처 업데이트 및 **고용됨** 변경.  |
| **휴직(배우자/동거인)** | **근로자 > 프로필 > 개인 정보 > 개인 연락처 > 피부양자 세부 정보 > 휴직** | 개인 연락처 생성 및 **휴직 적용 날짜** 가 정의됨. 개인 연락처 **휴직** 이 업데이트됨. 개인 연락처 **휴직 적용 날짜** 가 업데이트됨.  |
| **보장 변경(직위)** |<br><ul><li>**작업자 > 직위 할당 > 작업자 직위 할당**</li><li>**직위 > 직위**</li></ul>| 작업자 직위 지정 레코드에서 직위 변경. 직위의 작업자 할당 변경. |
| **보장 변경(급여)** |<br><ul><li>**작업자 > 보상 > 고정 계획**</li><li>**작업자 > 개인 정보 > 복리후생 연봉**</li></ul>| **복리후생 관리 > 인적 자원 공유 매개 변수 > 복리후생 > 복리후생 연봉** 이 활성화되지 않은 경우 **작업자 > 보상 > 고정 계획** 을 업데이트하면 생활 이벤트가 생성됩니다. **복리후생 관리 > 인적 자원 공유 매개 변수 > 복리후생 > 복리후생 연봉** 이 활성화된 경우 **작업자 > 개인 정보 > 복리후생 연봉** 을 업데이트하면 생활 이벤트가 생성됩니다. |
| **메디케어(직원/피부양자)** | **근로자 > 프로필 > 개인 정보 > 개인 연락처 > 피부양자 세부 정보 > 메디케어 적용 날짜** | 개인 연락처에 대한 **메디케어 유효 날짜** 를 추가하거나 업데이트하면 이 생활 이벤트가 생성됩니다. |
| **법원 명령 지원** | **작업자 > 프로필 > 개인 정보 > 개인 연락처 > 피부양자 > 법원 명령 지원**(QMSCO/QDRO) 및 적용 날짜 | 개인 연락처를 생성할 때 **법원 명령 지원** 이 **예** 인 경우 생활 이벤트가 생성됩니다. **법원 명령 지원** 또는 **법원 명령 만료 날짜** 를 업데이트하면 생활 이벤트가 발생합니다. |
| **사망** | **작업자 > 프로필 > 개인 정보 > 사망 날짜** | **사망 날짜** 가 입력되거나 업데이트됨. |
| **보험가능체 증거** | **작업자 > 작업자 > 버전 > 고용 이력 > 날짜 관리자 > 복리후생 세부 정보** | **보험가능체 증거** 가 **예** 로 설정됨. **보험가능체 증거 증빙 날짜** 가 정의됨. |
| **수혜자** | **작업자 > 프로필 > 개인 정보 > 개인 연락처** | 개인 연락처가 추가되고 **수혜자** 및 **적용 날짜** 필드가 채워짐. 개인 연락처는 **자녀**, **배우자**, **동거인**, **형제 자매**, **가족 연락처**, **기타 연락처**, **부모** 유형이어야 합니다. |
| **직원 메디케어** | **작업자 > 작업자 > 버전 > 고용 이력 > 날짜 관리자 > 복리후생 세부 정보** | **메디케어 자격** 이 **예** 로 설정됨. **메디케어 자격 날짜** 가 변경됨. |
| **생일** | **복리후생 관리 > 생활 이벤트 변경 처리** | 이러한 생활 이벤트는 **생활 이벤트 변경 처리** 에서 생성됩니다. 프로세스가 선택한 기간과 법인을 분석하여 연관된 작업자를 찾습니다. 마지막 생일을 계산하고 아직 생성되지 않은 경우 생일 이벤트를 생성합니다. |
| **작업자 자격 변경(미국에만 국한되지 않음)** |<br><ul><li>**작업자 > 고용**</li><li>**작업자 > 작업자 > 버전 > 고용 이력**</li></ul>| 다음의 경우 생활 이벤트를 만듭니다.<br><ul><li>새로운 고용을 만들고 이전 고용이 있으며 작업자 유형이 변경됨.</li><li>새 고용 세부 정보를 만들고 이전 고용 세부 정보가 있으며 고용 유형 또는 고용 범주가 변경됨.</li><li>고용 레코드 업데이트 및 다른 작업자 유형이 정의됨.</li><li>고용 세부 정보 레코드 업데이트 및 다른 고용 유형 또는 범주가 지정됨.</li></ul> |
| **새로운 자격 재정의(미국에만 국한되지 않음)** | **인적 자원 고급 > 복리후생 > 계획 > 복리후생 > 자격 규칙 재정의** | 생활 이벤트 처리 사용<br>작업자의 신규 복리후생 계획 자격 재정의를 생성하면 이 생활 이벤트가 트리거됩니다.<br>BenefitEligibilityRuleOverride.ValidFrom. |
| **자격 규칙 재정의 변경(미국에만 국한되지 않음)** | **인적 자원 고급 > 복리후생 > 계획 > 복리후생 > 자격 규칙 재정의** | 복리후생 계획 자격 재정의에서 **유효 기간 시작** 또는 **유효 기간 종료** 를 업데이트면 이 생활 이벤트가 트리거됩니다. |
| **자격 규칙 재정의 만료(미국에만 국한되지 않음)** | 복리후생 관리 > 생활 이벤트 변경 처리  | 이러한 생활 이벤트는 **생활 이벤트 변경 처리** 에서 생성됩니다. 프로세스가 선택한 기간과 법인을 분석하여 연관된 복리후생 계획 자격 재정의를 찾습니다. 재정의가 만료된 경우 생활 이벤트를 만듭니다. |
| **신규 복리후생 계획(미국에 국한되지 않음)** | **인적 자원 고급 > 복리후생 > 계획 > 신규** | 자격 옵션이 현재 계획에 추가됨. 자격 옵션이 연결된 새 계획이 추가됨.</br></br>HR 담당자는 이 경우 생활 이벤트 자격 처리를 실행해야 합니다. |
| **자격 규칙 변경(미국에만 국한되지 않음)** | **복리후생 관리 > 자격 규칙** | 생활 이벤트 자격 처리 사용. **BenefitEligibilityRule** 레코드의 **UseEmplCategory**, **UseEmplStatus** 또는 **UseEmplType** 값이 변경된 경우 기록됩니다. 변경된 규칙 또는 자격 기준에 대해 이미 존재하는 생활 이벤트 트랜잭션만 업데이트합니다. |
