# <a name="warehouse-performance-power-bi-content"></a>창고 실적 Power BI 콘텐츠

[!include [banner](../includes/banner.md)]

이 항목에서는 **창고 실적** Microsoft Power BI 콘텐츠에 포함된 항목에 대해 설명합니다. Power BI 보고서에 액세스하는 방법을 설명하고 콘텐츠를 구축하는 데 사용되는 데이터 모델 및 엔터티에 대한 정보를 제공합니다.

## <a name="overview"></a>개요

**창고 실적** Power BI 콘텐츠는 창고 및 운영 관리자가 중요한 인바운드, 아웃바운드 및 인벤토리 메트릭을 모니터링할 수 있도록 만들어졌습니다. 창고 관리, 제품 및 시스템의 기타 트랜잭션 데이터를 사용하고 창고 성능에 대한 집계 보기와 공급업체, 제품 그룹 및 제품, 사이트 및 창고에 대한 분석을 모두 제공합니다.

창고 관리자는 **창고 실적** Power BI 콘텐츠를 사용하여 다음 세 가지 영역을 측정할 수 있습니다.

- **인바운드 실적** – 공급업체가 고객 요구 사항에 대해 얼마나 잘 수행하고 있는지 측정(즉, 배송 성과 측정)하고 폐기 성과를 측정하여 일정 기간 동안 작업자 또는 품목과 관련된 문제를 식별할 수 있습니다. 공급업체 성능이 전반적인 폐기 성능에 어떤 영향을 미치는지 결정할 수 있도록 공급업체가 정시에 제공하는지, 일찍 또는 늦게 제공하는지 아는 것이 중요합니다. 합의된 날짜 외에 배송하는 공급업체는 예기치 않은 작업으로 인해 창고에 추가적인 부담을 줄 수 있으며 평균 보관 시간을 늘릴 수 있습니다.
- **배송 실적** – 창고가 고객에게 제시간에 완전히 배송되고 있는지 측정(즉, 아웃바운드 배송 및 배송 성과 측정)하여 제품, 사이트 또는 창고 또는 전용 고객과 관련된 문제를 식별할 수 있습니다. 특정 지역이나 마을로 늦게 배송되는 경우 운송 또는 계정 관리에 더 주의를 기울여야 할 수 있습니다.
- **위치 인벤토리 정확도** – 재고 정확성은 중요한 내부 창고 비즈니스 인텔리전스(BI)입니다. 일반적으로 얼마나 정확하게 계산하는지 확인하는 것이 매우 중요합니다. 그러나 올바른 위치에 항목을 저장하는 데 얼마나 정확한지 결정하고 불일치 데이터를 강조 표시하여 항목에 대한 더 나은 위치를 찾거나 특정 항목에 대한 총 계산을 시작할 수 있도록 하는 것도 중요합니다. (현재 새로운 아이템 기반 카운팅 기능은 핫픽스로 제공되고 있습니다.) 이 Power BI 콘텐츠를 사용하여 위치별 보유 재고 데이터의 정확성을 확인하는 경우 또한 상점에서 도난을 식별할 수 있습니다. 위치에 ERP(전사적 자원 관리) 데이터와 다른 현재고 수량이 있는지 여부도 확인할 수 있습니다. 그 위치가 너무 크거나 셀 수 없을 수도 있습니다. 또는 일부 물리적 위치가 좋지 않아 단일 유형의 항목을 보유 데이터와 동기화된 상태로 유지하기 어려울 수 있습니다.

## <a name="accessing-the-power-bi-content-pack"></a>Power BI 콘텐츠 팩에 액세스
**창고 실적** Power BI 콘텐츠는 **창고 실적** 페이지(**창고 관리** \> **문의 및 신고** \> **창고 실적 분석** \> **창고 실적**)에 표시됩니다.

| 배송 성능        | 선적된 라인                            | 주어진 기간 동안 배송된 총 배송 수입니다. |

## <a name="understanding-the-data-model-and-calculations"></a>데이터 모델 및 계산 이해
다음 데이터는 **창고 실적** Power BI 콘텐츠에서 보고서 페이지를 작성하는 데 사용됩니다. 이 데이터는 엔터티 저장소에 준비된 집계 측정값으로 표시됩니다. 엔터티 스토어는 분석에 최적화된 Microsoft SQL Server 데이터베이스입니다. 자세한 내용은 [엔터티 스토어와 Power BI 통합](power-bi-integration-entity-store.md)을 참조하세요.

다음 주요 집계 측정은 콘텐츠의 기준으로 사용됩니다.

| 보고서 페이지                 | 차트                                   | 테이블                                | 계산 설명 |
|-----------------------------|------------------------------------------|---------------------------------------|--------------------------|
| 인바운드 실적         | 총 처분                          | WHSWorkLine                           | 작업 유형이 **풋** 인 작업 라인 수입니다. |
| 인바운드 실적         | 평균 시간 처리                    | WHSWorkLine                           | 최대 작업 라인의 합계. 작업 라인 수로 나눈 시간 최대. 작업 라인이 최대인 시간. time은 작업 생성 날짜와 마감 날짜 사이의 최대 간격입니다. |
| 인바운드 실적         | 조기 수령                           | WHSWorkLine                           | 작업 생성 날짜가 사용된 납품 날짜보다 빠른 작업 라인의 수입니다. 배송 확정일자가 설정되지 않은 경우 기본 배송일을 사용하세요. |
| 인바운드 실적         | 정시에 접수됨                         | WHSWorkLine                           | 작업 생성 날짜가 사용된 납품 날짜와 같은 작업 라인의 수입니다. 배송 확정일자가 설정되지 않은 경우 기본 배송일을 사용하세요. |
| 인바운드 실적         | 늦게 수령                            | WHSWorkLine                           | 작업 생성 날짜가 사용된 납품 날짜보다 늦은 작업 라인의 수입니다. 배송 확정일자가 설정되지 않은 경우 기본 배송일을 사용하세요. |
| 인바운드 실적         | 공급업체 정시                        | WHSWorkLine                           | 일찍 수신됨, 정시에 수신됨 및 늦게 수신됨(이 표 앞부분의 설명 참조). |
| 인바운드 실적         | 재고 평균에 도킹(시간)   | WHSWorkLine                           | 평균 시간 처분(이 표 앞부분의 설명 참조). |
| 인바운드 실적         | 작업자가 버린 평균               | WHSWorkLine                           | 평균 시간 처분(이 표 앞부분의 설명 참조). |
| 인바운드 실적         | 공급업체별 평균 배치 시간          | WHSWorkLine                           | 평균 시간 처분(이 표 앞부분의 설명 참조). |
| 인바운드 실적         | 제품별 평균 배치 시간         | WHSWorkLine                           | 평균 시간 처분(이 표 앞부분의 설명 참조). |
| 위치 인벤토리 정확도 | 총 수                              | WHSWorkLineCycleCount                 | 절대 불일치 수량이 0(영) 이상인 경우 주기를 계산합니다. |
| 위치 인벤토리 정확도 | 불일치 비율                         | WHSWorkLineCycleCount                 | 불일치가 있는 주기 수를 총 수로 나눈 값입니다. 절대 불일치 수량이 0(영)보다 크면 주기 수에 불일치가 있는 것으로 간주됩니다. |
| 위치 인벤토리 정확도 | 불일치 없이 계산                | WHSWorkLineCycleCount                 | 절대 불일치 수량이 0(영)보다 큰 경우 주기를 계산합니다. |
| 위치 인벤토리 정확도 | 불일치가 있는 수                   | WHSWorkLineCycleCount                 | 절대 불일치 수량이 0(영)과 같은 경우 주기를 계산합니다. |
| 위치 인벤토리 정확도 | 시간 경과에 따라 계산된 항목                  | WHSWorkLineCycleCount                 | 불일치 없이 계산 및 불일치로 계산(이 표 앞부분의 설명 참조.) |
