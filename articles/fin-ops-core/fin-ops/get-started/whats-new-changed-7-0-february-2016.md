# <a name="whats-new-or-changed-in-dynamics-ax-70-february-2016"></a>Dynamics AX 7.0의 새로운 기능 또는 변경된 기능(2016년 2월)

[!include [banner](../includes/banner.md)]

이 문서에서는 Microsoft Dynamics AX 7.0의 새로 추가되거나 변경된 기능에 관해 설명합니다. 이 버전에는 플랫폼 및 애플리케이션 기능이 모두 포함되어 있으며 2016년 2월에 릴리스되었습니다.

## <a name="developer"></a>개발자

|------------------|------------------|-----------------|------------------------|
| 많은 장치에서 액세스할 수 있는 클라우드에서 웹 기반 솔루션을 만듭니다. | 사용할 수 없음 | Dynamics AX의 현재 버전은 웹 기반 클라이언트 및 클라이언트 프레임워크를 기반으로 합니다. | 최종 사용자에게 차세대 솔루션을 제공할 수 있습니다. |
| Microsoft Visual Studio를 사용하여 솔루션을 개발합니다. | Microsoft MorphX가 주요 개발 환경이지만 일부 개발은 Visual Studio에서 발생합니다. | Visual Studio는 유일한 개발 환경입니다. | Dynamics AX 2012 개념을 친숙하게 유지하고 Visual Studio 프레임워크와 패러다임에 완벽하게 적용합니다. 다른 .NET 언어 및 프로젝트와의 표준 상호 운용성을 가능하게 합니다. |
| 모든 기능에 대해 CIL(Common Intermediate Language)을 컴파일합니다. | X++는 p-코드로 컴파일됩니다. | 새로운 X++ 컴파일러는 모든 기능에 대해 CIL을 생성합니다. CIL은 다른 .NET 기반 언어에서 사용하는 것과 동일한 중간 언어입니다. | CIL은 더 빠르고 관리되는 DLL(동적 연결 라이브러리)의 클래스를 효율적으로 참조할 수 있으며 .NET 유틸리티의 대규모 도구 기반에서 실행할 수 있습니다. |
| Microsoft Dynamics AX 클라이언트에 BI(비즈니스 인텔리전스) 보고서 및 시각화 포함. | 사용할 수 없음 | 매우 직관적이고 유동적인 시각화를 생성합니다. | BI를 기반으로 한 의사결정 인사이트를 제공합니다. |
| Microsoft Office와 통합. | 사용할 수 없음 | 새로운 기능에는 Excel 데이터 커넥터 앱, **통합 문서 디자이너** 페이지, 내보내기 API 및 문서 관리가 포함됩니다. | 최종 사용자를 위한 생산성 솔루션을 만들 수 있습니다. |
| 빌드, 테스트 및 배포를 자동화합니다. | 부분적으로 사용 가능 | 개발자 및 빌드 VM을 사용하여 개발자 토폴로지를 배포합니다. Visual Studio Online(VSO)에서 빌드 VM을 자동 구성하여 모듈을 검색하고 빌드하고 테스트를 실행합니다. C\# 및 X++ 모듈 컴파일 및 참조가 지원됩니다. | 테스트 및 검증을 위한 비용과 노력을 줄여 개발자 생산성을 높입니다. |
| 오버레이 및 확장으로 사용자 정의합니다. | 확장은 사용할 수 없습니다. | Dynamics AX의 현재 버전에는 새로운 사용자 정의 모델이 있습니다. | Microsoft 또는 타사 Microsoft 파트너가 제공하는 모델 요소의 소스 코드 및 메타데이터를 사용자 지정할 수 있습니다. |
| X++ 및 최신 웹 프레임워크를 사용하여 새로운 컨트롤과 UI 요소를 빌드합니다. | 사용자 지정 컨트롤은 Microsoft ActiveX 및 WPF(Windows Presentation Foundation)와 같은 외부 프레임워크에 의존합니다. | 현재 버전에서 컨트롤을 빌드하는 것이 더 쉽습니다. X++ 프레임워크는 애플리케이션 동작 및 비즈니스 로직에 사용될 수 있으며 HTML/JavaScript 기반 클라이언트는 현대적인 시각화를 허용합니다. | 컨트롤은 Dynamics AX 기본 제공(OOB) 컨트롤과 같은 모양과 동작을 하도록 설계할 수 있습니다. |
| 새로운 도구를 사용하여 성능을 평가하고 조정합니다. | PerfSDK, Data Expansion Toolkit, Trace Parser WEb 앱 및 PerfTimer는 사용할 수 없습니다. | PerfSDK, Data Expansion Toolkit, Trace Parser 웹 앱 및 PerfTimer가 새로 추가되었습니다. | 소프트웨어 개발 키트(SDK)를 사용하면 단일 사용자 및 해당되는 경우 다중 사용자 테스트 실행에서 성능에 대한 모든 중요한 비즈니스 프로세스를 테스트하고 검증할 수 있습니다. Data Expansion Toolkit을 사용하면 마스터 데이터와 트랜잭션 데이터가 올바르게 확장되어야 하는 모든 성능 테스트를 올바르게 확장할 수 있습니다. Trace Parser를 사용하면 단일 사용자 성능 테스트 또는 다중 사용자 실행의 유효성을 검사할 수 있습니다. PerfTimer를 사용하면 쿼리나 특정 메서드 호출로 인해 성능 문제가 발생하는지 확인할 수 있습니다. 따라서 모든 것을 자세히 추적하고 분석할 필요가 없습니다. |
| OData를 사용하여 업데이트 가능한 뷰를 노출합니다. | 사용할 수 없음 | Dynamics AX의 현재 버전은 Dynamics AX 데이터에 대한 액세스를 가능하게 하는 공개 OData 서비스 엔드포인트를 광범위한 고객에게 일관된 방식으로 제공합니다. | 솔루션은 RESTful 서비스와 상호 작용하고 검색 가능한 방식으로 데이터를 공유하며 HTTP 스택 프로토콜을 사용하여 광범위한 통합을 가능하게 합니다. |
