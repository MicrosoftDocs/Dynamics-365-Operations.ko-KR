# <a name="feature-management-overview"></a>기능 관리 개요

[!include [banner](../../includes/banner.md)]


[!INCLUDE [PEAP](../../../../includes/peap-1.md)]

기능은 모든 릴리스에서 추가 및 업데이트됩니다. 기능 관리 환경은 각 릴리스에 제공된 기능 목록을 볼 수 있는 작업 공간을 제공합니다. 그런 다음 작업 공간을 사용하여 기능 설명서를 보고 기능을 활성화 또는 비활성화할 수 있습니다.

## <a name="the-feature-management-workspace"></a>기능 관리 작업 영역

대시보드에서 적절한 타일을 선택하여 **기능 관리** 작업 영역을 열 수 있습니다. 기능 관리 환경에서 지원하는 모든 릴리스의 기능 목록을 보여주는 페이지가 표시됩니다. 

기능 목록에는 다음 정보가 포함됩니다.

- **기능 이름** – 추가된 기능에 대한 설명입니다.
- **상태** – 기호는 기능이 켜져 있는지(확인 표시), 꺼져 있는지(공백), 켜져 있는지(시계), 필수(잠금), 켜기 전에 주의가 필요한지(경고 기호) 또는 켤 수 없음(X)을 나타냅니다. 표시된 설정은 모든 법인에 사용됩니다. 기능이 켜져 있는 경우에도 여전히 보안에 의해 제어됩니다. 따라서 이 기능은 보안 역할에 따라 액세스 권한이 있는 사용자만 사용할 수 있습니다. 또한 사용자가 액세스할 수 있는 법인에서만 사용할 수 있습니다.
- **활성화 날짜** – 기능이 켜졌거나 켜질 예정인 날짜입니다.
- **기능 추가됨** – 기능이 환경에 추가된 날짜입니다. 이 날짜는 월별 릴리스 주기 동안 환경을 업데이트할 때 자동으로 입력됩니다.
- **기능 상태** – 기능의 현재 수명 주기 상태: **프리뷰**, **릴리스됨**(공백으로 표시), **기본적으로 켜짐**, **필수**. 상태는 이 항목의 뒷부분에서 더 자세히 다룹니다. 
- **모듈** – 새 기능의 영향을 받는 모듈입니다.

> [!NOTE]
> **기능 상태** 열은 버전 10.0.21부터 포함됩니다.

기능을 선택하면 기능 목록 오른쪽의 세부 정보 창에 추가 정보가 나타납니다. 창 상단에는 기능 이름, 기능이 추가된 날짜, 기능의 영향을 받는 모듈 및 **자세한 정보** 링크가 표시됩니다. 기능에 대한 설명서를 보려면 이 링크를 선택합니다. 문서를 사용할 수 없는 경우 임시 페이지로 이동합니다. 세부 정보 창에는 기능에 대한 자신의 의견을 추가할 수 있는 **코멘트** 필드도 포함됩니다.

**기능 관리** 작업 공간에는 여러 탭이 있으며 각 탭에는 기능 목록이 표시됩니다.

- **새로운 기능** – 이 탭은 지난 월간 업데이트 이후에 추가된 모든 기능을 보여줍니다. 월별 업데이트를 건너뛴 경우 마지막으로 업데이트한 이후에 추가된 모든 새 기능이 탭에 표시됩니다. 최신 기능이 목록 맨 위에 나타납니다. 새로운 기능의 총 개수도 페이지 상단의 타일에 표시됩니다.
- **활성화되지 않음** – 이 탭은 켜져 있지 않은 모든 기능을 보여줍니다. 최신 기능이 목록 맨 위에 나타납니다. 또한 페이지 상단의 타일에는 현재 꺼져 있는 새 기능의 총 수가 표시됩니다.
- **예정** – 이 탭은 향후 켜질 예정인 모든 기능을 보여줍니다. 가장 빠른 예약 날짜가 있는 기능이 목록 맨 위에 나타납니다. 또한 페이지 상단의 타일에는 예정된 기능의 총 수가 표시됩니다.
- **모두** – 이 탭은 모든 기능을 보여줍니다. 최신 기능이 목록 맨 위에 나타납니다.

## <a name="feature-states"></a>기능 상태
기능은 기능 관리에 도입되는 것에서 궁극적으로 제품에서 필수가 되는 것까지 여러 상태 간에 전환될 수 있습니다. 이 섹션에서는 유효한 기능 상태에 대해 설명합니다.

### <a name="preview-features-optional"></a>미리 보기 기능(선택 사항)

제품 팀은 처음에 새 기능을 미리 보기 기능으로 시작하도록 결정할 수 있습니다. 미리보기 기능은 기본적으로 활성화되어 있지 않으며 선택 사항입니다. 소유 제품 팀은 성공적인 미리 보기 기간을 완료한 후 기능을 릴리스로 업데이트합니다.

> [!NOTE]
> 미리보기 기능은 특정 미리 보기 [이용약관](https://go.microsoft.com/fwlink/?linkid=2105274)이 적용됩니다. 

### <a name="released-features-optional"></a>출시된 기능(선택 사항)

이러한 기능의 **기능 상태** 열은 비어 있습니다. 처음에 릴리스된 기능으로 추가되는 기능은 기본적으로 켜져 있지 않으며 활성화하는 것은 선택 사항입니다. 미리 보기에서 업데이트된 기능은 활성화 상태를 유지합니다.

### <a name="on-by-default-features-optional"></a>기본 기능 켜기(선택 사항)

**기본적으로 켜짐** 으로 업데이트되는 기능은 기본적으로 켜져 있지만 비활성화할 수 있습니다. 비활성화할 수 있는 기능이 최소 6개월 동안 **출시됨** 상태로 있으면 다음 주요 릴리스에서 이 상태로 이동할 것으로 예상됩니다. **기본적으로 켜짐** 으로 전환되는 기능에서 릴리스에 대한 [새로운 소식](../whats-new-changed.md) 토픽에 전달될 것으로 예상됩니다. 업데이트는 소유 제품 팀에서 시작됩니다.

> [!NOTE]
> 이러한 기능은 자동으로 활성화되므로 조직에서 이러한 기능을 사용할 준비가 되었는지 또는 더 많은 시간이 필요한지 여부를 결정하는 것이 중요합니다. 시간이 더 필요한 경우 이러한 기능을 일시적으로 비활성화해야 할 수 있습니다. **기본적으로 켜짐** 으로 기능 전환은 일반적으로 기능이 **필수** 로 지정되기 전에 주요 릴리스에서 수행됩니다. 이 시점에서 기능을 비활성화할 수 있는 옵션이 없습니다. 

### <a name="mandatory"></a>필수

**필수** 는 기능의 예상 최종 상태입니다. 기능이 켜져 있으며 Microsoft에 문의하지 않고는 비활성화할 수 없음을 나타냅니다. 두 가지 주요 릴리스 이후에 선택적 기능이 필수가 될 것으로 예상됩니다. 중요한 기능은 예외적으로 필수 기능으로 도입될 수 있습니다.

## <a name="example-of-expected-feature-lifecycles"></a>예상되는 기능 수명 주기의 예

비활성화할 수 있고 4월 릴리스 이전 또는 일부로 릴리스 및 선택 사항으로 추가된 기능은 다음 10월 릴리스에서 **기본적으로 켜짐** 으로 전환될 예정입니다. 다음 연도 4월에 **필수** 가 될 것으로 예상됩니다.

비활성화할 수 있고 4월 릴리스 이전 또는 일부로 릴리스 및 선택 사항으로 추가된 기능은 다음 연도 4월에 **필수** 로 전환될 예정입니다.

## <a name="enable-a-feature"></a>기능 활성화

기능이 켜져 있지 않으면 **지금 활성화** 버튼이 세부 정보 창에 나타납니다. 이 버튼을 사용하여 기능을 활성화할 수 있습니다.

일부 기능은 활성화한 후에 비활성화할 수 없습니다. 켜려는 기능을 활성화할 수 없는 경우 경고를 받게 됩니다. 그 때 **취소** 를 선택하여 작업을 취소하고 기능을 비활성화된 상태로 둡니다. 그러나 **사용** 을 선택하여 이 기능을 활성화하면 나중에 비활성화할 수 없습니다.

일부 기능은 활성화하기 전에 추가 정보를 제공하는 메시지를 표시합니다. 이러한 기능은 노란색 경고 기호로 표시됩니다. 기능이 활성화될 때 어떤 일이 발생하는지 이해하려면 추가 정보를 주의 깊게 읽어야 합니다. 그러나 여전히 **사용** 을 선택하여 기능을 활성화할 수 있습니다.

일부 기능은 조치가 취해질 때까지 기능을 활성화할 수 없다는 메시지를 표시합니다. 이러한 기능은 빨간색 X 기호로 표시됩니다. 기능을 활성화하기 전에 설명에 설명된 작업을 수행해야 합니다. 예를 들어 구성 키가 비활성화될 때까지 기능을 사용할 수 없는 경우 먼저 구성 키를 비활성화한 다음 기능 관리로 돌아가 기능을 활성화해야 합니다.

기능이 활성화되면 세부 정보 창의 **자세히 알아보기** 링크 아래에 메시지가 나타납니다. 이 메시지는 기능이 활성화되었음을 나타내거나 기능이 활성화되도록 예약된 미래 날짜를 나타냅니다. 기능 목록에서 기능을 선택할 때마다 나타납니다.

향후 활성화될 예정인 기능은 **예정** 탭에 표시됩니다. 일괄 처리는 시스템 날짜가 나타내는 시간대를 기준으로 지정된 날짜의 자정에 활성화합니다.

## <a name="reschedule-a-feature"></a>기능 일정 변경

기능이 향후 활성화되도록 예약된 경우 **일정** 버튼이 세부 정보 창에 나타납니다. 이 버튼을 사용하여 **활성화 날짜** 값을 다른 날짜로 변경합니다.

1. 일정을 변경할 예약된 기능을 선택한 다음 세부 정보 창에서 **일정** 을 선택합니다.
2. 표시되는 대화 상자에서 **날짜 활성화** 필드에 기능을 활성화해야 하는 새 날짜를 지정합니다.
3. **사용** 을 선택하여 기능을 다시 예약하거나 **사용 안 함** 을 선택하여 일정을 취소합니다.

## <a name="disable-a-feature"></a>기능 비활성화

기능이 켜져 있으면 **비활성화** 버튼이 세부 정보 창에 나타납니다. 이 버튼을 사용하여 기능을 비활성화할 수 있습니다. 기능을 비활성화할 수 없는 경우 **비활성화** 버튼을 사용할 수 없습니다. 

기능이 비활성화되면 세부 정보 창의 **자세히 알아보기** 링크 아래에 메시지가 나타납니다. 이 메시지는 기능이 활성화되지 않았음을 나타냅니다. 기능 목록에서 기능을 선택할 때마다 나타납니다. 활성화되지 않은 기능은 **활성화되지 않음** 탭에 표시됩니다.

## <a name="features-that-must-be-enabled"></a>활성화해야 하는 기능

경우에 따라 업데이트를 수행할 때 자동으로 활성화되어야 하는 중요한 기능이 제공됩니다. 이 기능은 **활성화 날짜** 필드에 지정된 날짜에 자동으로 활성화됩니다. 이러한 기능의 경우 세부 정보 창의 **자세히 알아보기** 링크 아래에 메시지가 나타납니다. 이 메시지는 기능이 활성화되었음을 나타내거나 기능이 활성화되는 미래 날짜를 나타냅니다. 기능 목록에서 기능을 선택할 때마다 나타납니다.

## <a name="enable-all-features"></a>모든 기능 활성화

**모두 사용** 버튼을 선택하여 모든 기능을 활성화할 수 있습니다. 

**모두 활성화** 를 선택하면 다음 정보를 제공해야 하는 옵션이 나타납니다.

- 활성화하기 전에 확인이 필요한 모든 기능의 목록입니다. 목록의 기능을 활성화하려면 **확인이 필요한 기능 활성화** 단추에 **예** 를 선택합니다.
- 활성화할 수 없는 모든 기능의 목록이 표시됩니다. 해당 기능은 활성화되지 않습니다.

활성화할 수 있는 모든 기능이 활성화됩니다. 기능이 미래에 활성화되도록 이미 예약되어 있는 경우 일정은 변경되지 않습니다. 

## <a name="enable-all-features-automatically"></a>자동으로 모든 기능 활성화

모든 새 기능을 자동으로 활성화하려면 작업 영역 제목 아래의 드롭다운 목록을 사용하여 새 기능이 추가될 때 발생하는 작업을 변경할 수 있습니다.

- **새로운 기능을 자동으로 활성화** 를 선택하여 환경에 추가될 때 모든 새로운 기능을 자동으로 활성화합니다.
- 환경에 추가될 때 모든 가능한 새로운 기능을 기본적으로 사용하지 않아야 하는 경우 **새로운 기능을 자동으로 활성화 안 함** 을 선택합니다.

모든 기능을 자동으로 활성화하면 **모두 사용** 버튼을 클릭할 때 활성화되는 모든 기능이 활성화됩니다. 확인이 필요한 기능이나 조치가 취해질 때까지 활성화할 수 없는 기능은 활성화되지 않습니다.

## <a name="check-for-updates"></a>업데이트 확인

각 업데이트 후에 기능이 환경에 추가됩니다. 그러나 **업데이트 확인** 단추를 클릭하여 업데이트를 수동으로 확인할 수 있습니다. 업데이트 후 시스템에 추가된 모든 기능은 기능 목록에 추가됩니다. 예를 들어 릴리스 후 플라이트 기능이 활성화된 경우 업데이트를 확인할 수 있으며 기능이 목록에 추가됩니다.

## <a name="assigning-roles"></a>역할 할당

**기능 관리** 작업 공간은 시스템 관리자와 기능 관리자 역할 또는 기능 뷰어 역할에 할당된 사용자가 열 수 있습니다. 이 두 가지 역할은 기능 관리 경험을 지원하기 위해 만들어졌습니다. 기능 관리자 역할의 사용자는 모든 기능을 켜거나 끌 수 있습니다. 또한 기능의 **코멘트** 필드를 업데이트할 수 있습니다. 기능 뷰어 역할의 사용자는 **기능 관리** 작업 공간을 보기만 할 수 있습니다. 기능을 켜거나 끌 수 없습니다.

기능 관리자 역할 및 기능 뷰어 역할은 사용자가 가지고 있는 기존 보안을 재정의하지 않습니다. 사용자가 기능을 켜고 끌 수 있는지 여부만 제어합니다. 기능 자체에 대한 액세스를 제공하지 않습니다.

## <a name="features-that-use-configuration-keys"></a>구성 키를 사용하는 기능

기능이 구성 키를 사용하지만 구성 키가 켜져 있지 않은 경우 **기능 관리** 작업 공간의 사용 가능한 기능 목록에 해당 기능이 표시되지 않습니다. 구성 키를 켠 후에는 **업데이트 확인** 메뉴 항목을 사용하여 기능 목록을 업데이트해야 합니다. 그러면 해당 기능이 기능 목록에 나타납니다.

구성 키를 끄면 기능이 기능 목록에서 제거되지 않습니다.

## <a name="data-entities"></a>데이터 엔터티

이름이 **기능 관리** 인 데이터 엔터티를 사용하면 한 환경에서 기능 관리 설정을 내보낸 다음 다른 환경으로 가져올 수 있습니다. 이 엔터티는 기존 기능만 업데이트합니다. 엔터티의 비즈니스 로직은 또한 가져오기가 완료되면 **기능 관리** 작업 영역에서 사용되는 동일한 규칙이 적용되도록 합니다. 예를 들어 가져오는 동안 날짜를 제거하여 필수 기능 설정을 재정의할 수 없습니다.

다음 예에서는 **기능 관리** 엔터티를 사용하여 데이터를 가져올 때 발생하는 사항을 설명합니다.

- **활성화됨** 필드의 값을 **네** 로 변경하면 기능이 활성화되고 **날짜 활성화** 필드는 현재 날짜로 설정됩니다.
- **활성화됨** 필드의 값을 **아니요** 로 변경하거나 **EnableDate** 필드를 비워 두면 기능이 비활성화되고 **날짜 활성화** 필드는 선택 취소됩니다. 필수 기능이나 활성화한 후에 비활성화할 수 없는 기능은 비활성화할 수 없습니다.
- **날짜 활성화** 필드의 값을 미래 날짜로 변경하면 해당 날짜에 기능이 예약됩니다.
- **활성화됨** 필드의 값을 **네** 로 변경하고 **EnableDate** 필드의 값을 미래 날짜로 변경하면 기능이 해당 날짜로 예약됩니다. 
- **활성화됨** 필드의 값을 **아니요** 로 변경하고 **EnableDate** 필드의 값을 미래 날짜로 변경하면 기능이 해당 날짜로 예약됩니다.
- 기능이 활성화되어 있고 미래 날짜로 설정된 **EnableDate** 필드를 추가하면 이 기능은 활성화된 상태로 유지됩니다. 기능을 다시 예약하려면 **활성화됨** 필드의 값을 **아니요** 로 변경해야 합니다.

## <a name="feature-management-and-flighting"></a>기능 관리 및 플라이트

기능 관리를 통해 각 릴리스에서 제공되는 기능을 제어할 수 있습니다. 플라이팅을 사용하면 Microsoft 팀은 제한된 수의 고객에게 기능을 릴리스하여 모든 고객에게 영향을 주지 않고 해당 기능을 테스트하고 검증할 수 있도록 합니다. 기능 관리는 기능의 비행을 제어하지 않습니다.

## <a name="using-feature-management-to-turn-on-isv-features-or-custom-features"></a>기능 관리를 사용하여 ISV 기능 또는 사용자 지정 기능 켜기

기능 관리는 현재 ISV(독립 소프트웨어 공급업체)의 기능 및 사용자 지정 기능에 사용할 수 없습니다. 그러나 Microsoft는 기능 관리를 향상시키기 위해 더 많은 기능을 추가하고 있습니다. 이러한 개선 사항이 완료되면 Microsoft는 모든 기능에서 기능 관리를 사용할 수 있도록 하고 기능을 사용하도록 업데이트하기 위한 지침을 제공합니다.

## <a name="frequently-asked-questions-faq"></a>자주 묻는 질문(FAQ)

### <a name="when-are-features-added-removed-or-changed"></a>기능은 언제 추가, 제거 또는 변경됩니까? 
기능은 소유 제품 팀의 코드 변경을 통해 추가, 제거 및 변경됩니다. 이러한 변경 사항을 수신하려면 환경을 업데이트해야 합니다.

### <a name="does-a-feature-become-mandatory-automatically"></a>기능이 자동으로 필수가 됩니까? 
아니요, 기능이 자동으로 필수로 설정되지 않습니다. 소유 제품 팀은 코드를 변경해야 합니다.

### <a name="why-isnt-there-a-specific-mandatory-enabled-date"></a>특정 '필수 활성화 날짜'가 없는 이유는 무엇입니까? 
업데이트 릴리스 타이밍은 가변적이며 환경 업데이트 타이밍은 가변적이며 고객은 일부 업데이트를 건너뛰도록 선택할 수 있습니다. 결과적으로 구체적인 날짜를 결정하기 어렵습니다. 

### <a name="wheres-the-documentation-for-features-that-are-mandatory"></a>필수 기능에 대한 문서는 어디에 있습니까? 
이 설명서는 각 Dynamics 365 응용 프로그램 팀에서 제공합니다. 종종 이러한 기능은 [클라이언트 기능 상태 업데이트](/dynamics365-release-plan/2021wave1/finance-operations/finance-operations-crossapp-capabilities/updates-client-feature-states) 또는 [제거되거나 더 이상 사용되지 않는 기능](../../../dev-itpro/migration-upgrade/deprecated-features.md)에 언급됩니다. 

### <a name="is-there-an-in-product-notification-or-signal-that-a-feature-is-going-to-be-mandatory-enabled"></a>기능이 필수로 활성화될 것이라는 제품 내 알림 또는 신호가 있습니까? 
기능을 필수로 만드는 것과 관련된 알림 메커니즘은 현재 존재하지 않습니다.

### <a name="do-features-ever-get-enabled-without-the-customer-knowing-about-it"></a>고객이 알지 못하는 사이에 기능이 활성화된 적이 있습니까? 
예, 다음과 같은 상황에서 고객이 모르게 기능을 활성화할 수 있습니다.
- 기능이 **기본적으로 켜짐** 으로 이동됩니다. 이 상태에서는 기능을 계속 비활성화할 수 있습니다. 
- 기능이 **필수** 로 업데이트됩니다. 이 변경 사항은 주요 릴리스와 함께만 발생합니다. 중요한 기능은 업데이트 시 예외적으로 **필수** 로 이동할 수 있습니다.

### <a name="what-is-feature-flighting-and-how-does-it-relate-to-feature-management"></a>기능 비행이란 무엇이며 기능 관리와 어떤 관련이 있습니까? 
기능 비행은 Microsoft가 제어하는 실시간 켜기/끄기 스위치입니다. 기능 관리에서 제공하는 고객 제어와는 별개입니다. 
- 비공개 미리 보기 기능은 비행이 시작될 때까지 기능 관리에 나열되지 않습니다. 생산 과정에서 고객은 특별 프로그램의 일부에 동의해야 해당 프로그램이 발생합니다.
- 공개 미리 보기 및 릴리스(일반적으로 사용 가능) 기능은 제외되지 않는 한 기능 관리에 나열됩니다. 중요한 문제가 발견되고 일반적으로 고객별 작업이 되는 경우 기능 해제는 제품 팀의 최후 수단 옵션으로 간주됩니다.

### <a name="do-features-ever-get-flighted-off-without-the-customer-knowing-about-it"></a>고객이 알지 못하는 사이에 기능이 플라이트 오프된 적이 있습니까? 
예, 기능이 기능에 영향을 미치지 않는 환경의 기능에 영향을 미치는 경우 기본적으로 활성화할 수 있습니다.

### <a name="how-can-feature-enablement-be-checked-in-code"></a>기능 활성화는 코드에서 어떻게 확인할 수 있습니까?
**FeatureStateProvider** 클래스에서 **isFeatureEnabled** 메서드를 사용하여 기능 클래스의 인스턴스를 전달합니다. 예: 

```xpp
if (FeatureStateProvider::isFeatureEnabled(BatchContentionPreventionFeature::instance()))
```

### <a name="how-can-feature-enablement-be-checked-in-metadata"></a>기능 활성화는 메타데이터에서 어떻게 확인할 수 있습니까?
**FeatureClass** 속성을 사용하여 일부 메타데이터가 기능과 연결되었음을 나타낼 수 있습니다. 기능에 사용되는 클래스 이름은 다음과 같이 사용해야 합니다. **BatchContentionPreventionFeature**. 이 메타데이터는 해당 기능에서만 볼 수 있습니다. **FeatureClass** 속성은 메뉴, 메뉴 항목, 열거형 값 및 테이블/보기 필드에서 사용할 수 있습니다.

### <a name="what-is-a-feature-class"></a>피쳐 클래스란 무엇입니까?
기능 관리의 기능은 *기능 클래스* 로 정의됩니다. 기능 클래스는 **IFeatureMetadata를 구현** 하고 기능 클래스 속성을 사용하여 기능 관리 작업 영역에서 자신을 식별합니다. **기능 상태 제공자** API 및 메타데이터에서 **기능 클래스** 속성을 사용하여 코드에서 활성화 여부를 확인할 수 있는 사용 가능한 피쳐 클래스의 여러 예가 있습니다. 예: 

```xpp
[ExportAttribute(identifierStr(Microsoft.Dynamics.ApplicationPlatform.FeatureExposure.IFeatureMetadata))]
internal final class BankCurrencyRevalGlobalEnableFeature implements IFeatureMetadata
```

### <a name="what-is-the-ifeaturelifecycle-implemented-by-some-feature-classes"></a>일부 피쳐 클래스에서 구현된 IFeatureLifecycle은 무엇입니까?
IFeatureLifecycle은 기능 수명 주기 단계를 표시하기 위한 Microsoft 내부 메커니즘입니다. 기능은 다음과 같습니다.
- `PrivatePreview` - 보이려면 플라이트가 필요합니다.
- `PublicPreview` - 기본적으로 표시되지만 기능이 미리 보기에 있다는 경고와 함께 표시됩니다.
- `Released` - 완전히 해제되었습니다.
