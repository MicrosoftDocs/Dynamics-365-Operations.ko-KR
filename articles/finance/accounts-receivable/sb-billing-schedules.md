# <a name="billing-schedule-overview"></a>청구 일정 개요

[!include [banner](../includes/banner.md)]
[!include [preview banner](../includes/preview-banner.md)]

**청구 일정** 페이지에서 청구 일정을 만들고, 삭제하거나 편집할 수 있습니다. 청구 일정 목록을 검토할 수도 있습니다. 청구 일정을 만들 때 해당 일정의 기본값은 연결된 청구 그룹에 의해 결정됩니다. 추가 정보는 **반복 계약 청구 매개 변수** 페이지에서 설정됩니다.

## <a name="create-a-billing-schedule"></a>청구 일정 만들기

청구 일정을 만들려면 다음 단계를 따르세요.

1. **청구 일정** 페이지에서 **새로 만들기** 를 선택합니다.
2. **청구 일정 만들기** 대화 상자의 **청구 일정 그룹** 필드에서 청구 일정 그룹을 찾습니다.
3. **고객 계정** 필드에서 고객 계정을 선택합니다.
4. **시작 날짜** 필드에서 시작 날짜를 선택한 다음 **기간 수** 필드에 기간 수를 입력합니다. **종료 날짜** 필드는 입력한 기간 수에 따라 업데이트됩니다. **청구 종료 날짜** 필드를 업데이트하면 **기간 수** 필드가 **0**(영)으로 업데이트됩니다.
5. **확인** 을 선택합니다.
6. **청구 일정** 페이지의 **일반** 탭에 있는 **설명** 필드에 청구 일정에 대한 설명을 입력합니다.
7. **마일스톤 템플릿** 필드에서 **마일스톤 청구** 의 마일스톤 템플릿을 선택합니다.

**청구 계정** 및 **통화 코드** 와 같은 필드가 고객의 정보로 업데이트됩니다.

**청구 빈도** 및 **청구 간격** 필드가 선택한 청구 일정 그룹에 따라 자동으로 설정됩니다.

8. 개별적으로 송장을 생성하려면 **개별적으로 송장 발부** 옵션을 **예** 로 설정합니다.
9. 최종 청구 기간 이후에 자동으로 청구 일정을 갱신하려면 **자동으로 갱신** 옵션을 **예** 로 설정한 후 **갱신당 추가할 라인** 필드를 설정합니다.

**매개 변수** 필드는 **반복 계약 청구 매개 변수** 페이지의 값에 따라 자동으로 설정됩니다.

10. 청구 일정 금액을 비례 배분하려면 **일부 기간 비례 배분** 옵션을 **예** 로 설정합니다.
11. 청구 일정 세부 정보 라인을 월말에 맞추려면 **월에 맞춤** 옵션을 **예** 로 설정합니다.
12. **계약 시작 날짜** 및 **계약 종료 날짜** 필드에 계약의 시작 날짜와 종료 날짜를 입력합니다. 이러한 날짜는 참고용입니다.

**지불** 필드는 고객의 고객 지불 정보를 보여줍니다. 라인 항목이 보류 중이거나 종료된 경우 지불 정보를 변경할 수 없습니다.

> [!NOTE]
> 항목별로 송장을 통합할 때 **결제 조건**, **방법** 및 **청구 일정** 필드의 값이 일치해야 합니다. 그렇지 않으면 송장을 통합할 수 없습니다.

13. **주소** 탭에서 **배송 주소** 및 **청구지 주소** 필드를 검토하고 업데이트할 수 있습니다.
14. **연락처 정보** 탭에서 최종 사용자 계정을 청구 일정과 연결할 수 있습니다.
15. **연락처 주소** 필드에서 연락처, 이메일 주소 및 인터넷 주소를 입력할 수 있습니다.
16. 파트너 수수료 정보를 추적하려면 **파트너 계정** 및 **파트너 수수료율** 필드를 설정합니다. 이러한 필드는 참고용입니다.
17. **총계** 탭에서는 청구 일정에 대해 계산된 다양한 총계를 볼 수 있습니다.
18. **보류** 탭에서 보류가 해제되었을 때 청구 일정이 보류된 시기에 관한 감사 정보를 볼 수 있습니다.
19. **해지** 탭에서는 청구 일정에 적용되거나 삭제된 해지 내역을 볼 수 있습니다.
20. **저장** 을 선택합니다.
21. **청구 일정 라인** 빠른 탭에서 **라인 추가** 를 선택합니다.
22. **항목 번호** 필드에서 항목 번호를 선택합니다. 추가된 항목이 수익 분할의 상위 항목인 경우 라인은 하위 항목으로 자동 업데이트됩니다. 수익 분할의 상위 항목만 편집할 수 있습니다. 그러면 모든 하위 항목이 그에 따라 업데이트됩니다.
23. **항목 유형** 필드에서 항목 유형을 선택합니다.
24. 시작 날짜와 종료 날짜를 업데이트합니다.
25. 송장이 생성되기 전에 **청구 빈도** 필드를 업데이트하여 청구 빈도를 변경할 수 있습니다. 청구 일정에 대한 첫 번째 송장이 생성된 후에는 청구 빈도를 변경할 수 없습니다.
26. **단위** 필드에서 항목의 측정 단위를 선택합니다.
27. **가격 책정 방법** 필드에서 항목의 가격 책정 방법을 선택합니다.

**단가** 필드는 인벤토리에서 자동으로 설정됩니다. 단, 가격 책정 방식을 **고정** 으로 변경하면 업데이트할 수 있습니다.

## <a name="remove-a-line-item"></a>라인 항목 제거

청구 일정에서 항목을 제거하려면 다음 단계를 따르세요.

1. **청구 일정** 페이지의 **일정 번호** 필드에서 수정할 청구 일정의 번호를 선택합니다.
2. **청구 일정 라인** 빠른 탭에서 삭제할 라인을 선택한 다음 **제거** 를 선택합니다.
3. **저장** 을 선택합니다.

이 항목의 나머지 부분에서는 **청구 일정 라인** 빠른 탭의 라인에 사용할 수 있는 작업 및 세부 정보에 관해 설명합니다.

## <a name="billing-schedule-line-actions"></a>청구 일정 라인 작업

**청구 일정 라인** 빠른 탭의 버튼을 사용하면 개별 라인에 작업을 적용할 수 있습니다.

| 버튼 | 설명 |
|--------|-------------|
| 라인 추가 | 청구 일정에 라인 추가 |
| 항목 목록에서 추가 | 목록에서 항목을 선택하여 청구 일정에 여러 항목을 추가합니다. |
| 제거 | <p>청구 일정에서 선택한 라인을 제거합니다.</p><p>수익 분할의 일부인 항목의 경우 상위 항목만 제거할 수 있습니다. 그러면 연결된 모든 하위 항목이 자동으로 제거됩니다.</p> |
| 청구 세부 정보 보기 | 청구 세부 정보를 봅니다. |
| 취소 | <p>선택한 라인을 취소합니다. 이 버튼은 선택한 라인이 **활성** 상태인 경우에만 사용할 수 있습니다.</p><p>수익 분할의 일부인 항목의 경우 상위 항목만 취소할 수 있습니다.</p> |
| 취소 제거 | <p>선택한 라인에서 취소를 제거합니다. 이 버튼은 선택한 라인이 **취소됨** 상태인 경우에만 사용할 수 있습니다.</p><p>수익 분할의 일부인 항목의 경우 상위 항목에서만 취소를 제거할 수 있습니다.</p> |
| 보류 | <p>선택한 회선을 보류할 세부 정보를 선택합니다.</p><p>수익 분할의 일부인 항목의 경우 상위 항목만 보류할 수 있습니다.</p> |
| 보류 제거 | <p>선택한 라인에서 보류를 제거합니다.</p><p>수익 분할의 일부인 항목의 경우 상위 항목에서만 보류를 제거할 수 있습니다.</p> |
| 에스컬레이션 및 할인 | 수익 분할이 **0 금액** 할당 방법을 사용하는 상위 항목을 제외하고 수익 분할의 일부인 항목에는 이 버튼을 사용할 수 없습니다. |
| 이연 | <p>선택한 라인에 대한 이연 옵션을 입력합니다.</p><p>수익 분할의 상위 항목에는 이 버튼을 사용할 수 없습니다.</p> |
| 마일스톤 할당 | <p>선택한 라인에 대한 마일스톤 할당 정보를 검토하고 갱신합니다.</p><p>이 버튼은 청구 일정 항목이 마일스톤 항목인 경우에만 사용할 수 있습니다.</p> |
| 지원 및 갱신 | <p>선택한 라인에 대한 지원 및 갱신 정보를 검토하고 업데이트합니다.</p><p>이 버튼은 청구 일정 항목이 지원 또는 갱신 항목인 경우에만 사용할 수 있습니다.</p> |
| 차원 표시 | **청구 일정 라인** 그리드에 표시되는 차원 열을 표시하거나 숨깁니다. |
| 단가 계산 | <p>고객이 계약 금액을 분할 지불할 수 있도록 품목의 단가를 계산합니다(예: 매일, 매월, 매 분기, 매 반기, 매년). 계약 가격과 가격 빈도를 선택할 수 있습니다.</p><p>이전 계약 가격 및 빈도, 새 계약 가격 및 빈도, 변경한 사용자, 변경 날짜 및 시간 등 변경 내용에 대한 감사 추적을 볼 수 있습니다.</p> |
| 맞춤 날짜 | <p>갱신 항목에 대한 맞춤 날짜를 지정합니다.</p><p>**항목 그룹** 필드에서 항목 그룹을 선택하면 모든 항목은 청구 일정의 항목 그룹에 있는 첫 번째 항목의 맞춤 날짜를 사용합니다. **항목 그룹** 필드가 비어 있는 경우 모든 항목에 사용할 맞춤 날짜를 지정할 수 있습니다.</p><p>이 버튼은 **청구 빈도** 필드가 **매년** 으로 설정된 경우에만 사용할 수 있습니다.</p> |
| 미청구 수익 | <p>미청구 수익 기능을 사용하도록 항목을 설정합니다. 그런 다음 항목에 대해 청구되지 않은 수익 및 청구되지 않은 할인 계정을 지정할 수 있습니다.</p><p>이 버튼은 **항목 유형** 필드가 **표준** 으로 설정된 항목에만 사용할 수 있습니다. 기존 청구 일정 또는 송장이 이미 생성된 청구 일정 라인에는 사용할 수 없습니다.</p> |
| 수익 분할 하위 항목 추가 | <p>판매 주문에 추가할 하위 항목을 선택합니다.</p><p>이 버튼은 수익 분할의 상위 항목에만 사용할 수 있습니다.</p> |
| 상세 가격 책정 옵션 | 항목에 대한 가격 책정 옵션을 편집합니다. |

## <a name="billing-schedule-line-details"></a>청구 일정 라인 세부 정보

**청구 일정 라인** 빠른 탭에서 라인을 선택하면 해당 라인에 대한 특정 세부 정보를 볼 수 있습니다. 이러한 세부 정보는 여러 탭으로 나뉩니다.

### <a name="general-tab"></a>일반 탭

**일반** 탭에서 다음 정보를 사용할 수 있습니다.

| 필드 또는 섹션 | 설명 |
|------------------|-------------|
| 사용량 | <p>이 섹션에서는 사용 항목에 대한 정보를 제공합니다. 여기에는 다음 필드가 포함됩니다.</p><ul><li>**사용 식별자** – 계량 또는 사용 항목의 식별자입니다.</li><li>**읽기 옵션** – 사용 읽기 옵션으로 **읽기** 또는 **소비** 입니다.</li><li>**예상 소비량** – 송장이 생성되지 않은 기간이 있는 사용 항목에 대한 예상 소비를 지정합니다. **청구 세부 정보** 페이지에서 예상 소비량에 대한 결제 세부 정보 라인을 검토할 수 있습니다.</li></ul> |
| 외부 참조\* | 이 섹션에는 외부 참조 정보를 지정할 수 있는 **외부** 및 **라인 번호** 필드가 있습니다. |
| 마일스톤 | <p>이 섹션에서는 마일스톤 항목에 대한 정보를 제공합니다. 여기에는 항목 완료 날짜를 표시하는 **예상 완료 날짜** 필드가 포함됩니다.</li></ul> |
| 텍스트 | 라인에 대한 코멘트. 텍스트는 고객 또는 법인의 기본 언어로 번역됩니다. |
| 항목 그룹 | 라인 항목의 항목 그룹입니다. |
| 맞춤 날짜 | 청구 일정에 대한 맞춤 날짜입니다. |

\* **송장 생성** 페이지에서 항목별로 송장을 통합할 때 **외부 참조** 필드가 일치해야 합니다. 한 글자라도 다른 경우에는 송장에 통합되지 않습니다. **외부 참조** 섹션의 어느 필드에도 유효성 검사가 수행되지 않으며 **라인 번호** 필드의 값은 양의 정수여야 합니다.

### <a name="address-tab"></a>주소 탭

**주소** 탭에서 다음 정보를 사용할 수 있습니다.

| 필드 | 설명 |
|-------|-------------|
| 배송 주소 | <p>라인 항목의 배송 주소를 선택합니다. 기본 배송 주소는 **주소** 빠른 탭의 기본 배송 주소입니다.</p><p>주소를 변경할 때 다음 주소 옵션을 선택할 수 있습니다.</p><ul><li>**주소** – 현재 고객의 주소를 선택합니다.</li><li>**사용 중** – 현재 고객이 현재 사용 중인 주소를 선택합니다.</li><li>**다른 주소** – 고객 기록의 주소를 선택합니다.</li></ul><p>수익 분할을 사용하는 항목의 경우 상위 항목의 주소만 편집할 수 있습니다. 하위 항목의 주소는 상위 항목의 주소와 일치하며 별도로 편집할 수 없습니다.</p> |
| 청구지 주소 | <p>라인 항목의 청구지 주소를 선택합니다. 기본 배송 주소는 **주소** 빠른 탭의 기본 배송 주소입니다. 사용 가능한 주소의 목적에 맞게 필요에 따라 주소를 변경할 수 있습니다.</p><ul><li>주소 중 **송장** 용도가 없는 경우 기본 청구지 주소는 목적과 관계없이 고객의 기본 주소입니다.</li><li>하나 이상의 주소에 **송장** 용도가 있는 경우 기본 청구지 주소는 가장 최근에 입력한 주소입니다.</li><li>하나 이상의 주소가 **송장** 의 목적이고 송장 주소 중 하나가 기본 주소로 설정된 경우 기본 청구지 주소는 **송장** 의 목적을 가진 주소이며 기본 주소로 설정됩니다.</li><li>수익 분할을 사용하는 항목의 경우 상위 항목의 주소만 편집할 수 있습니다. 하위 항목의 주소는 상위 항목의 주소와 일치하며 별도로 편집할 수 없습니다.</li></ul> |

### <a name="renewals-tab"></a>갱신 탭

**갱신** 탭에서 다음 정보를 사용할 수 있습니다.

| 필드 | 설명 |
|-------|-------------|
| 자동으로 갱신 | <p>청구 일정 라인이 다음 청구 기간에 대해 자동으로 갱신되는지를 나타내는 값:</p><ul><li>**예** – 청구 일정 라인이 송장을 생성할 때 다음 청구 기간에 대해 자동으로 갱신됩니다.</li><li>**아니요** – 청구 일정 라인이 자동으로 갱신되지 않습니다. 청구 일정을 수동으로 갱신해야 합니다.</li></ul> |
| 갱신당 추가할 라인 | 청구 일정 갱신에 추가할 라인 수. |

또한 **갱신** 탭에서 다음 버튼을 사용할 수 있습니다.

| 버튼 | 설명 |
|--------|-------------|
| 미청구 수익 분개장 항목 감사 | 미청구 수익 기능을 사용하는 항목에 대한 모든 변경 사항을 봅니다. |
| 갱신 기간 추가 | 항목에 대한 갱신 기간을 추가합니다. 새로운 갱신 기간의 시작 날짜는 이전 기간의 종료 날짜 다음 날입니다. **갱신 종료 날짜**, **이연 시작 날짜**, **이연 종료 날짜**, **항목 수량** 및 **단가** 필드는 업데이트할 수 있습니다. |
| 갱신 기간 수정 | <p>갱신 기간을 수정합니다. 초기 기간의 경우 초기 분개장 항목이 생성되기 전에 이연 시작 및 종료 날짜를 변경할 수 있습니다. 후속 기간의 경우 시작 날짜를 변경할 수 없습니다. 항상 이전 기간 종료 후 다음 날짜입니다.</p><p>수정하는 기간 이후에 갱신 기간이 있는 경우 기간의 날짜를 변경할 수 없습니다. 이 경우 갱신 품목에 대한 **수량** 및 **단가** 필드만 업데이트할 수 있습니다.</p><p>예를 들어 세 가지 용어가 있습니다. <ul><li>첫 번째 기간은 이미 시작되었으므로 변경할 수 없습니다.</li><li>두 번째 기간은 수량과 단가만 변경할 수 있습니다.</li><li>세 번째 기간은 시작 날짜를 제외한 모든 값을 변경할 수 있습니다. 또한 **템플릿에서 일정 예약** 옵션을 사용하면 청구되지 않은 수익 항목에 대한 템플릿을 기반으로 이연 일정을 만들 수 있습니다. 이 옵션이 **예** 로 설정된 경우 **템플릿** 필드에서 이연 템플릿을 선택하고 필요에 따라 이연 시작 및 종료 날짜를 변경합니다. 후속 갱신 기간은 같은 이연 템플릿을 사용합니다. 그러나 이연 템플릿은 변경할 수 있습니다.</p></li></ul> |
