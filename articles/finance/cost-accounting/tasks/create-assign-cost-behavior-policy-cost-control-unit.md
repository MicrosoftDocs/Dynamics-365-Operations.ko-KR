# <a name="create-and-assign-a-cost-behavior-policy-to-a-cost-control-unit"></a>비용 행동 정책을 만들고 비용 제어 장치에 할당

[!include [banner](../../includes/banner.md)]

비용 행동은 비용을 고정 또는 변동으로 분류하는 것입니다. 정책과 해당 규칙이 비용 제어 장치에 할당되어야 정책이 유효합니다. 이 절차를 사용하여 정책을 만든 다음 비용 제어 장치에 정책을 할당합니다.


## <a name="create-a-cost-behavior-hierarchy"></a>비용 행동 계층 만들기
1. 원가 회계 > 차원 > 차원 계층으로 이동합니다.
2. 새로 만들기를 클릭합니다.
3. 만들기를 클릭합니다.
4. 차원 계층 이름 필드에 '비용 동작 계층'을 입력합니다.
5. 차원 필드에서 값을 입력하거나 선택합니다.
    * 비용 요소를 선택합니다.  
6. 저장을 클릭합니다.
7. 계층 보기를 클릭합니다.
8. 새로 만들기를 클릭합니다.
9. Node 이름 필드에 값을 입력합니다.
    * 고정 비용을 입력합니다.  
10. 트리에서 '비용 행동 계층'을 선택합니다.
11. 새로 만들기를 클릭합니다.
12. Node 이름 필드에 값을 입력합니다.
    * 변동 비용을 입력합니다.  
13. 저장을 클릭합니다.
14. 트리에서 '비용 행동 계층\고정 비용'을 선택합니다.
15. 새로 만들기를 클릭합니다.
16. 목록에서 선택한 행을 표시합니다.
17. 원본 차원 구성원 필드에서 값을 입력하거나 선택합니다.
    * 차원 구성원의 범위는 간격을 포함할 수 있지만 구성원은 겹칠 수 없습니다.  
18. 대상 차원 구성원 필드에서 값을 입력하거나 선택합니다.
    * 차원 구성원의 범위는 간격을 포함할 수 있지만 구성원은 겹칠 수 없습니다.  
19. 트리에서 '비용 행동 계층\변동 비용'을 선택합니다.
20. 새로 만들기를 클릭합니다.
21. 목록에서 선택한 행을 표시합니다.
22. 원본 차원 구성원 필드에서 값을 입력하거나 선택합니다.
    * 차원 구성원의 범위는 간격을 포함할 수 있지만 구성원은 겹칠 수 없습니다.  
23. 대상 차원 구성원 필드에서 값을 입력하거나 선택합니다.
    * 차원 구성원의 범위는 간격을 포함할 수 있지만 구성원은 겹칠 수 없습니다.  
24. 저장을 클릭합니다.

## <a name="create-the-policy-and-rules"></a>정책 및 규칙 만들기
1. 원가 회계 > 정책 > 비용 행동 정책으로 이동합니다.
2. 새로 만들기를 클릭합니다.
3. 정책 이름 필드에 값을 입력합니다.
4. 비용 요소 차원 계층 필드에서 값을 입력하거나 선택합니다.
    * 방금 생성한 정책 계층을 선택합니다.  
5. 비용 개체 차원 계층 필드에서 값을 입력하거나 선택합니다.
    * 조직을 선택합니다.  
6. 저장을 클릭합니다.
7. 새로 만들기를 클릭합니다.
8. 목록에서 선택한 행을 표시합니다.
9. 비용 요소 차원 계층 노드 필드에서 값을 입력하거나 선택합니다.
    * 계층을 확장하여 변동 비용을 선택합니다.  
10. 비용 개체 차원 계층 노드 필드에서 값을 입력하거나 선택합니다.
    * 기본적으로 변수 백분율은 100%입니다.  
11. 비용 제어 장치에 대한 정책 할당을 클릭합니다.
12. 새로 만들기를 클릭합니다.
13. 목록에서 선택한 행을 표시합니다.
14. 회계 적용 날짜 필드에 날짜를 입력합니다.
    * 규칙은 날짜부터 유효하며 새 버전이 생성되면 사용자 또는 시스템이 규칙을 만료시킬 수 있습니다.  
15. 비용 제어 장치 필드에 값을 입력하거나 선택합니다.
16. 저장을 클릭합니다.
