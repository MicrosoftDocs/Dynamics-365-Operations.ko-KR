# <a name="fixed-asset-transaction-options"></a>고정 자산 거래 옵션

[!include [banner](../includes/banner.md)]

이 항목에서는 고정 자산 거래를 생성하는 다양한 방법을 설명합니다.

지급 계정, 수취 계정, 조달 및 소싱, 총계정원장과의 통합을 위해 고정 자산을 설정할 수 있습니다. 내부적으로 사용하려는 경우 재고 관리의 항목을 고정 자산으로 전송할 수도 있습니다.

## <a name="accounts-payable"></a>지급 계정
분개장 바우처 페이지에서 고정 자산 거래를 입력할 수 있습니다. 이 페이지는 송장 분개장 페이지에서 열 수 있습니다. 송장 승인 분개장 페이지에서 분개장 바우처 페이지를 열 수도 있습니다. 상쇄 계정 유형 필드에서 고정 자산을 선택합니다. 그런 다음 상쇄 계정 필드에서 고정 자산 번호를 선택합니다. 고정 자산 탭에서 거래 유형 및 장부 필드에 값을 입력합니다.

## <a name="accounts-receivable"></a>수취 계정
자유 텍스트 송장 페이지에서 고정 자산 거래를 입력할 수 있습니다.  자유 텍스트 송장 페이지의 송장 라인 그리드에서 라인 항목을 선택합니다. 라인 세부 정보 빠른 탭을 클릭합니다. 처분 거래에 대한 고정 자산 번호 및 장부를 입력합니다. 자유 텍스트 송장의 경우 고정 자산 거래 유형은 항상 처분 - 판매입니다.

## <a name="procurement-and-sourcing"></a>조달 및 소싱
구매 주문 페이지에서 고정 자산 거래를 입력할 수 있습니다. 구매 주문을 생성하는 데 필요한 정보를 입력하고 확인을 클릭합니다. 구매 주문 페이지에서 라인 세부 정보 빠른 탭을 클릭합니다. 그런 다음 고정 자산 탭에서 고정 자산에 대한 정보를 입력합니다. 

기존 고정 자산에 대한 취득 거래를 게시하려면 고정 자산 번호, 장부 및 거래 유형을 지정합니다. 이러한 정보가 누락되면 고정 자산을 게시할 수 없습니다. 신규 고정 자산에 대한 취득 거래를 게시하려면 신규 고정 자산입니까? 옵션을 선택한 다음 신규 자산을 할당할 고정 자산 그룹을 선택합니다. 그러나 품목이 표준 원가 재고 모델을 사용하는 재고 모델 그룹에 있는 경우 라인에 대해 고정 자산 필드를 사용할 수 없습니다. 또한 고정 자산 매개 변수 페이지에 정의된 옵션에 따라 구매 모듈에서 취득 거래를 게시할 수 있는지 결정됩니다. 

고정 자산을 취득하는 데 구매 주문이나 재고-고정 자산 분개장을 사용하면 재고 가치가 영향을 받습니다.

## <a name="general-ledger"></a>총계정원장
모든 고정 자산 거래 유형은 일반 분개장 페이지에 게시될 수 있습니다. 고정 자산의 분개장을 사용하여 고정 자산 거래를 게시할 수도 있습니다.

### <a name="options-for-entering-fixed-asset-transaction-types"></a>고정 자산 거래 유형 입력 옵션


| 트랜잭션 유형                    | 모듈                   | 옵션                                   |
|-------------------------------------|--------------------------|-------------------------------------------|
| 취득, 취득 조정 | 고정 자산             | 고정 자산, 재고-고정 자산   |
|                                     | 총계정원장           | 일반 분개장                           |
|                                     | 지급 계정         | 송장 분개장, 송장 승인 분개장 |
|                                     | 조달 및 소싱 | 구매 주문                            |
| 감가상각                        | 고정 자산             | 고정 자산                              |
|                                     | 총계정원장           | 일반 분개장                           |
| 처분                            | 고정 자산             | 고정 자산                              |
|                                     | 총계정원장           | 일반 분개장                           |
|                                     | 수취 계정      | 무료 텍스트 송장                         |

감가상각 거래 유형 분개장 라인을 수동으로 생성하거나 데이터 엔터티를 통해 가져오는 경우 고정 자산의 감가상각 기간 동안 나머지 값이 업데이트되지 않습니다. 감가상각 제안 프로세스를 사용하여 분개장 라인을 생성하면 나머지 값이 업데이트됩니다.

자세한 내용은 [고정 자산 통합](fixed-asset-integration.md)을 참조하세요.

시스템은 감가상각을 같은 기간에 두 번 게시하지 못하게 합니다. 예를 들어, 두 사용자가 1월의 감가상각 제안을 별도로 만드는 경우 첫 번째 사용자의 감가상각이 첫 번째 분개장에 게시됩니다. 두 번째 사용자가 두 번째 분개장에 감가상각을 게시하면 시스템은 감가상각이 마지막으로 실행된 날짜를 확인하고 같은 기간 동안 두 번 감가상각을 게시하지 않습니다.

### <a name="transactions-that-require-a-different-voucher-number"></a>다른 바우처 번호가 필요한 거래

다음 고정 자산 거래는 서로 다른 바우처 번호를 사용합니다.

- 자산에 대한 추가 취득이 이루어지고 "전기손익추정" 감가상각이 계산됩니다.
- 자산이 분할됩니다.
- 폐기 시 감가상각을 계산하는 매개 변수가 켜진 후 자산이 처분됩니다.
- 자산의 서비스 날짜가 취득 날짜 이전입니다. 따라서 감가상각 조정이 게시됩니다.

> [!NOTE]
> 거래를 입력할 때 모든 거래가 같은 고정 자산에 적용되는지 확인하세요. 총계정원장의 **분개장 이름** 페이지에서 **신규 바우처** 필드가 **하나의 바우처 번호만** 으로 설정되어 있어도 둘 이상의 고정 자산이 포함된 바우처는 게시되지 않습니다. 바우처에 고정 자산을 두 개 이상 포함된 경우 "바우처당 고정 자산 거래는 하나만 가능합니다"라는 메시지가 표시되고 바우처를 게시할 수 없습니다.
