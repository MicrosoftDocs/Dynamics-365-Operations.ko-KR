# <a name="shipment-auto-updates"></a>배송 자동 업데이트

[!include [banner](../includes/banner.md)]

자동 업데이트 배송 기능은 화물이 창고로 출고된 후 배송과 관련된 로드 라인의 수량(증가 및 감소 모두)을 자동으로 업데이트합니다. 이 기능은 선적의 로드 라인이나 웨이브에서 로드가 처리될 때까지 켜진 상태로 유지됩니다. 이를 사용하면 창고 작업이 생성될 때까지 수동 개입 없이 주문 업데이트가 자동으로 창고로 흐를 수 있습니다.

자동 업데이트 배송 기능을 사용하지 않는 경우 창고 작업이 생성될 때까지 수량 감소만 자동으로 흐릅니다. 사용자는 라인을 수동으로 업데이트하거나 삭제해야 하며 주문 수량이 증가하거나 새 주문 라인이 추가되면 라인을 다시 릴리스해야 합니다. 자동 업데이트 배송 기능을 사용하여 기업은 관련 배송 및 로드가 주문 라인 업데이트를 반영하지 않을 것이라는 걱정 없이 창고에 업데이트를 원활하게 제공할 수 있습니다.

자동 업데이트 배송 기능은 판매 주문 라인과 이전 주문 라인 모두에 적용되며 특정 창고에 대해 설정되어 있습니다. 따라서 회사는 필요에 따라 창고에 서로 다른 자동 업데이트 배송 정책을 적용할 수 있습니다. 기본적으로 수량 감소에 대한 자동 업데이트 배송 정책은 창고 관리 프로세스를 사용하는 모든 창고에 적용됩니다. 이 기본 정책 설정을 사용하면 창고 작업이 생성될 때까지 수량 감소만 자동으로 배송으로 흘러들어갑니다. 이 동작은 자동 업데이트 배송 기능이 도입되기 전에 사용된 동작과 유사합니다.

## <a name="main-elements-of-the-functionality"></a>기능의 주요 요소

자동 업데이트 배송 기능은 판매 주문 라인 또는 이전 주문 라인이 변경될 때 로드 라인의 수량을 변경해야 하는지 여부를 결정하기 위해 주로 배송 상태에 의존합니다. 또한 새 로드 라인이 기존 로드에 자동으로 추가되어야 하는 시기를 결정하기 위해 주로 배송 상태에 의존합니다. 배송 상태가 **웨이브됨** 이상인 경우 자동 업데이트가 발생하지 않습니다.

자동 업데이트에는 웨이브 상태도 고려됩니다. 로드 라인과 관련된 웨이브가 **유지됨**, **실행 중**, **출고됨**, **피킹됨** 또는 **배송됨** 이고 사용자가 로드 라인의 수량을 줄이려고 하면(판매 주문 라인 또는 이전 주문 라인의 수량 감소를 통해) 다음 오류 메시지가 표시됩니다. "예약에 의존하는 작업이 생성되었으므로 예약을 삭제할 수 없습니다." 또한, 웨이브가 앞서 언급한 웨이브 상태 중 하나일 때, 사용자가 판매 주문 라인 또는 이전 주문 라인의 수량을 늘려 로드 라인 수량을 간접적으로 늘리려고 하면 로드 라인의 수량은 자동으로 증가하지 않습니다. 이 경우 로드 라인을 수동으로 업데이트해야 합니다.

## <a name="scenarios"></a>시나리오

자동 업데이트 배송 기능은 새 주문 라인 추가, 주문 라인의 수량 증가, 주문 라인의 수량 감소 및 주문 라인 제거의 네 가지 시나리오를 지원합니다.

- **주문 라인 추가** - **창고** 페이지의 **창고** 빠른 탭에 있는 **자동 업데이트 배송** 필드(**창고 관리 \> 설정 \> 창고 \> 창고**)가 **항상** 으로 설정되어 있고 주문에 대한 배송이 존재하고 판매 주문에 대한 로드가 이미 생성된 후 새 주문 라인이 판매 주문 또는 이전 주문에 추가되는 경우 기존 로드는 업데이트되지 않습니다. 기존 로드에 대한 참조가 없는 새 로드 라인이 생성되어 기존 배송과 연결됩니다. 새 라인이 로드에 추가되고 해제됩니다.
- **주문 라인의 수량 늘리기** - **자동 업데이트 배송** 필드가 **항상** 으로 설정되고 주문에 대한 선적이 존재하고 판매 주문에 대한 로드가 이미 생성된 후 기존 판매 주문 라인 또는 이전 주문 라인의 수량이 증가하는 경우 로드 라인은 주문 라인과 동일한 수량만큼 증가합니다. 로드가 해제되었지만 작업이 생성되지 않은 경우 로드 라인은 주문 라인과 동일한 수량만큼 증가합니다.
- **주문 라인의 수량 감소** - **자동 업데이트 배송** 필드가 **항상** 또는 **수량 감소 시** 로 설정되어 있고 주문에 대한 선적이 존재하고 판매 주문에 대한 로드가 이미 생성된 후 기존 판매 주문 라인 또는 이전 주문 라인의 수량이 감소하는 경우 연결된 로드 라인의 수량이 일치하도록 업데이트됩니다. 해당 로드 라인의 수량은 이미 주문 라인의 새 수량과 같거나 적습니다. 이 경우 로드 라인은 영향을 받지 않습니다. 로드가 해제되었지만 작업이 생성되지 않은 경우 로드 라인의 수량이 주문 라인의 새 수량과 이미 같거나 작지 않는 한 연관된 로드 라인의 수량이 일치하도록 업데이트됩니다. 이 경우 로드 라인은 영향을 받습니다.
- **주문 라인 제거** - **자동 업데이트 배송** 필드가 **항상** 또는 **수량 감소 시** 로 설정되고 사용자가 로드 라인이 존재하는 주문 라인을 제거하려고 하면 오류 메시지가 표시됩니다.

## <a name="example-scenario"></a>예시 시나리오

이 시나리오의 경우 데모 데이터가 설치되어 있어야 하며 **USMF** 데모 데이터 회사를 사용해야 합니다.

### <a name="turn-on-the-auto-update-shipment-functionality"></a>자동 업데이트 배송 기능 켜기

자동 업데이트 배송 기능을 켜려면 다음 단계를 따르세요.

1. **창고 관리 \> 설정 \> 창고 \> 창고** 로 이동합니다.
2. 창고 **24** 를 선택합니다.
3. **창고** 빠른 탭의 **자동 업데이트 배송** 필드에서 **수량 감소 시** 의 값을 **항상** 을 ㅗ변경합니다.

값을 **항상** 으로 변경한 후, 판매 주문 라인 및 이전 주문 라인의 수량 증가 또는 감소, 신규 라인 추가는 이전에 언급된 업데이트 제약 조건을 고려하여 선택한 창고의 선적 및 로드에 반영됩니다.

### <a name="change-the-wave-template-so-that-load-lines-arent-automatically-processed"></a>로드 라인이 자동으로 처리되지 않도록 웨이브 템플릿 변경

로드 라인을 자동으로 처리하지 않도록 웨이브 템플릿을 구성하려면 다음 단계를 따르세요.

1. **창고 관리 \> 설정 \> 웨이브 \> 웨이브 템플릿** 으로 이동합니다.
2. 웨이브 템플릿 **24 배송 기본값** 을 선택합니다.
3. **편집** 을 선택합니다.
4. **일반** 빠른 탭에서 **웨이브 생성 자동화** 옵션을 **예** 로 설정하고, 다른 모든 옵션이 **아니요** 로 설정되어 있는지 확인합니다.

웨이브 생성 프로세스의 일부로 작업이 자동으로 생성 및 해제되지 않도록 하는 것이 중요합니다. 판매 주문 라인에 대해 생성된 로드 라인과 관련된 작업이 생성된 후 판매 주문 라인의 수량이 변경되더라도 로드 라인은 더 이상 자동으로 업데이트되지 않습니다.

### <a name="create-a-sales-order"></a>판매 주문 만들기

판매 주문을 생성하려면 다음 단계를 따르세요.

1. **영업 및 마케팅 \> 판매 주문 \> 모든 판매 주문** 으로 이동합니다.
2. 고객 **US-003** 을 선택합니다.
3. 품목 번호 **A0001** 의 라인을 만듭니다.
4. 수량 **10** 을 입력합니다. (창고 **24** 를 사용 중인지 확인합니다.)
5. **저장** 을 선택합니다.
6. 작업 창의 **창고** 탭에 있는 **작업** 그룹에서 **창고로 출고** 를 선택합니다. 선적 및 웨이브가 생성됩니다.

이전 절차에서 웨이브 템플릿을 변경했기 때문에 하중이나 워크가 생성되지 않습니다. 배송 상태는 **시작됨** 이고 웨이브 상태는 **생성됨** 입니다.

### <a name="decrease-the-quantity-on-a-sales-order-line"></a>판매 주문 라인의 수량 감소

판매 주문 라인의 수량을 줄이려면 다음 단계를 따르세요.

1. **영업 및 마케팅 \> 판매 주문 \> 모든 판매 주문** 으로 이동합니다.
2. 방금 창고로 출고한 판매 주문을 선택합니다.
3. 판매 주문 라인을 선택합니다. **수량** 필드에서 값을 **10** 에서 **8** 로 변경합니다.
4. 판매 주문 라인에서 **창고 \> 배송 세부 정보** 를 선택합니다. **배송 세부 정보** 페이지의 **로드 라인** 빠른 탭에서 수량은 판매 주문 라인의 변경 사항을 반영합니다.

### <a name="increase-the-quantity-on-a-sales-order-line"></a>판매 주문 라인의 수량 증가

판매 주문 라인의 수량을 늘리려면 다음 단계를 따르세요.

1. **영업 및 마케팅 \> 판매 주문 \> 모든 판매 주문** 으로 이동합니다.
2. 이전에 창고로 출고한 판매 주문을 선택합니다.
3. 라인 수량을 **8** 에서 **12** 로 변경합니다.
4. **저장** 을 선택합니다.
5. **모든 판매 주문** 페이지로 돌아가서 판매 주문을 다시 선택합니다.
5. 작업 창에 있는 **창고** 탭의 **관련 정보** 그룹에서 **배송 세부 정보** 를 선택합니다. **배송 세부 정보** 페이지의 **로드 라인** 빠른 탭에서 수량은 판매 주문 라인의 변경 사항을 반영합니다.

적재 라인의 수량은 8개에서 12개로 늘렸지만 자동 예약이 켜져 있지 않으면 8개만 예약되어 있습니다. 기존 발송물에 추가된 수량은 예약되지 않았기 때문에 이 시점에서 웨이브가 처리되면 예약 없이 이미 예약된 수량에 대해서만 작업이 생성됩니다.

> [!NOTE]
> 주문 라인의 수량이 감소할 때 로드 라인의 수량이 주문 라인의 새 수량과 이미 같거나 작으면 영향을 받지 않습니다. 주문 라인의 수량이 증가하면 로드 라인은 주문 라인과 동일한 수량만큼 증가합니다. 주문 라인의 수량이 로드 라인의 수량과 다른 경우 차이는 그대로 유지됩니다.

### <a name="add-a-sales-order-line"></a>판매 주문 라인 추가

판매 주문 라인을 추가하려면 다음 단계를 따르세요.

1. **영업 및 마케팅 \> 판매 주문 \> 모든 판매 주문** 으로 이동합니다.
2. 이전에 창고로 출고한 판매 주문을 선택합니다.
3. 품목 번호 **A0002** 의 라인을 만듭니다.
4. **수량** 필드에 **10** 을 입력합니다. (창고 **24** 를 사용 중인지 확인합니다.) 새 라인은 기존 발송물에 자동으로 추가됩니다.
5. **저장** 을 선택합니다.
6. **모든 판매 주문** 페이지로 돌아가서 판매 주문을 다시 선택합니다.
7. 작업 창에 있는 **창고** 탭의 **관련 정보** 그룹에서 **배송 세부 정보** 를 선택합니다. **배송 세부 정보** 페이지의 **로드 라인** 빠른 탭에서 두 번째 로드 라인을 확인합니다.

기존 배송에 방금 추가한 판매 주문 라인이 예약되지 않았기 때문에 이 시점에서 웨이브가 처리되면 첫 번째 판매 주문 라인과 첫 번째 로드 라인의 수량에 대해서만 작업이 생성됩니다.

### <a name="process-a-wave"></a>웨이브 처리

웨이브를 처리하려면 다음 단계를 따르세요.

1. **창고 관리 \> 아웃바운드 웨이브 \> 배송 웨이브 \> 모든 웨이브** 로 이동합니다.
2. 이전에 생성한 웨이브를 선택합니다.
3. 작업 창의 **웨이브** 탭에 있는 **웨이브** 그룹에서 **프로세스** 를 선택합니다.

웨이브가 처리되고 로드 라인에 예약된 수량에 대한 작업이 생성됩니다. 배송 상태가 **시작됨** 에서 **웨이브됨** 으로 업데이트됩니다. 배송 상태가 **웨이브됨** 으로 업데이트됨에 따라 라인 수량의 감소 또는 증가, 판매 주문에 신규 라인 추가와 같은 변경 사항은 웨이브된 선적과 연관된 기존 로드 라인에 영향을 미치지 않습니다.

배송 상태가 **웨이브됨** 이상인 경우 판매 주문 라인의 수량 업데이트가 배송과 연결된 로드 라인에 반영되거나 검증되지 않습니다. 로드 라인의 수량 변경은 로드 라인에서 직접 이루어져야 합니다.

로드 라인에 대한 작업이 생성되고 예약이 완료된 후 유효성 검사가 수행됩니다. 그런 다음 판매 주문 라인의 수량 감소가 작업 라인 예약에 대해 검증됩니다.
