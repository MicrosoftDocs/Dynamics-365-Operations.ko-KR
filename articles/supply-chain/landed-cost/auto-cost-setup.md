# <a name="auto-costs-setup"></a>자동 비용 설정

[!include [banner](../../includes/banner.md)]

**자동 비용** 페이지를 사용하여 다양한 비용 영역(예: 항해, 선적 컨테이너, Folio, 구매 주문, 품목 또는 이전 주문 라인)에 대한 비용 규칙을 설정할 수 있습니다. 규칙과 사용자가 비용 영역 중 하나에 대한 레코드를 생성할 때 선택한 필드를 기반으로 시스템은 비용을 계산하고 자동으로 추가합니다. 따라서 사용자가 수동으로 비용을 추가할 필요가 없습니다.

자동 비용으로 작업하려면 **상륙 비용 \> 원가계산 설정 \> 자동 비용** 으로 이동합니다. 그런 다음 이 항목의 나머지 부분에 설명된 대로 자동 비용 규칙을 설정합니다.

## <a name="work-with-cost-areas"></a>비용 영역 작업

자동 비용은 무역 협정이나 자동 잡비처럼 작동합니다. 각 자동 비용 레코드는 특정 비용 수준에 속합니다. **자동 비용** 페이지의 목록 창에서 **비용 영역** 을 사용하여 작업하려는 비용 영역을 선택합니다. 목록 창에는 현재 선택한 비용 영역에 속하는 자동 비용만 표시됩니다. 생성하는 모든 자동 비용은 현재 선택한 비용 영역에 속합니다.

비용 영역을 통해 시스템은 비용 영역의 구매 주문 라인 전체에 비용을 배분할 수 있습니다. 예를 들어, 하나의 배송 컨테이너에 대한 비용은 해당 배송 컨테이너의 모든 제품에 값을 할당합니다. 두 개 이상의 배송 컨테이너가 있는 경우 각 배송 컨테이너에 대해 동일한 비용 유형을 지정할 수 있습니다. 따라서 컨테이너 유형을 사용하여 올바른 자동 비용을 찾을 수 있습니다.

## <a name="buttons-on-the-action-pane"></a>작업 창의 단추

다음 표에서는 **자동 비용** 페이지의 작업 창에서 사용할 수 있는 버튼에 대해 설명합니다.

| 단추 | 설명 |
|---|---|
| 편집 | 기존 자동 비용을 수정합니다. |
| 신규 | 자동 비용을 생성합니다. |
| 삭제 | 자동 비용을 삭제합니다. |
| 다음에서 복사 | 기존 레코드를 기반으로 하는 새 자동 비용 레코드를 만듭니다. 그런 다음 새 레코드를 자유롭게 편집할 수 있습니다. 이 버튼을 사용하면 새 자동 비용을 빠르게 생성할 수 있습니다. |
| 디스플레이 치수 | 보고 있는 비용 트랜잭션에 대해 표시되는 인벤토리 차원을 선택할 수 있는 대화 상자를 엽니다. 확인란을 선택 취소하면 비용 거래에 대해 더 적은 수의 재고 차원이 표시됩니다. 따라서 거래에 대한 세부 정보가 덜 표시됩니다. 자동 비용에 인벤토리 차원이 지정된 경우 지정된 인벤토리 차원이 항해 항목에 사용되는 경우에만 자동 비용이 적용됩니다. |

## <a name="settings-on-the-header"></a>헤더 설정

헤더 섹션의 설정 조합에 따라 특정 자동 비용이 항해에 추가되는 시기와 방법이 결정됩니다. 자동 비용의 세부 정보가 해당 비용 영역 헤더의 세부 정보와 일치하는 경우에만 항해, 선적 컨테이너 또는 Folio에 자동 비용이 적용됩니다. 일치하는 모든 자동차 비용의 합계가 항해의 예상 상륙 비용을 결정합니다. 이 비용은 선박 또는 항해의 모든 항목에 대해 배분됩니다.

다음 표에서는 헤더 섹션에 나타날 수 있는 모든 필드에 대해 설명합니다. 그러나 사용 가능한 특정 필드는 현재 선택한 비용 영역 및 디스플레이 차원에 따라 다릅니다.

| 필드 | 설명 |
|---|---|
| **계정 코드** | <p>다음 값 중 하나를 선택합니다.</p><ul><li>**테이블** – 비용 규칙은 특정 공급업체에만 적용됩니다.</li><li>**그룹** – 비용 규칙은 특정 공급업체 그룹에 적용됩니다. 시스템은 공급업체 레코드와 연결된 [공급업체 비용 유형 그룹](costing-parameters-setup.md)을 조회합니다.</li><li>**모두** – 비용 규칙은 모든 공급업체에 적용됩니다. |
| **배송 회사** | 비용 규칙이 적용되는 공급업체 또는 공급업체 그룹을 선택합니다. 이 필드는 **계정 코드** 필드에서 *테이블* 또는 *그룹* 을 선택한 경우에만 사용할 수 있습니다. |
| **관세사** | 비용 규칙이 적용되는 공급업체 또는 공급업체 그룹을 선택합니다. 이 필드는 **계정 코드** 필드에서 *테이블* 또는 *그룹* 을 선택한 경우에만 사용할 수 있습니다. |
| **항목 코드** | <p>다음 값 중 하나를 선택합니다.</p><ul><li>**테이블** – 비용 규칙은 특정 항목에만 적용됩니다.</li><li>**그룹** – 비용 규칙은 특정 항목 그룹에 적용됩니다. 시스템은 항목 레코드와 연결된 [항목 비용 유형 그룹](costing-parameters-setup.md)을 조회합니다.</li><li>**모두** – 비용 규칙은 모든 항목에 적용됩니다.|
| **항목 관계** | 비용 규칙이 적용되는 항목 또는 항목 그룹을 선택합니다. 이 필드는 **항목 코드** 필드에서 *테이블* 또는 *그룹* 을 선택한 경우에만 사용할 수 있습니다. |
| **배송 모드** | 비용 규칙이 적용되는 배송 모드(예: 항공 또는 해상)를 선택합니다. |
| **컨테이너 유형** | 상품이 배송될 배송 컨테이너의 유형입니다. 자동 비용은 컨테이너 유형 설정이 항해의 컨테이너 유형과 일치하는 경우에만 항해에 적용할 수 있습니다. |
| **출발 항구** 및 **도착 항구** | 항해의 출발지("출발") 항구와 목적지("끝") 항구. (일부 선적 컨테이너는 항해가 여러 항구에서 멈출 수 있기 때문에 "도착" 항구가 다를 수 있습니다.) 자동 비용은 자동 비용 설정의 "출발" 및 "도착" 항구가 항해의 "출발" 및 "도착" 항구와 일치하는 경우에만 적용할 수 있습니다. |
| **자동 비용 번호** | 자동 비용 규칙의 고유 식별자입니다. 이 필드의 값은 **상륙 비용 매개 변수** 페이지에 설정된 번호 시퀀스에 따라 자동으로 생성됩니다. |
| **인벤토리 차원** | 일부 비용 영역에서는 헤더에 하나 이상의 항목 차원(예: 크기, 색상, 창고 및 배치 번호)을 포함할 수 있습니다. 작업 창에서 **디스플레이 차원** 을 선택하여 포함해야 하는 차원을 선택합니다. |

## <a name="settings-on-the-lines-fasttab"></a>라인 빠른 탭 설정

**라인** 빠른 탭에서 항해 중 구매 주문 라인에 비용이 적용될 때 사용할 수 있는 각 통화에 대한 행을 추가합니다. 

품목 및 구매 주문의 경우 시스템은 각 구매 주문 라인의 통화를 **라인** 빠른 탭에 지정된 통화와 일치시킵니다. 일치하는 라인 또는 항목에 대해 설정된 비용 값만 적용됩니다. 라인이나 항목의 통화에 대해 설정된 라인이 없으면 자동 비용은 해당 라인이나 항목에 적용되지 않습니다.

다른 유형의 비용 영역의 경우 **라인** 빠른 탭의 모든 라인이 헤더에 설정된 값과 일치하는 각 항해, 선적 컨테이너, 폴리오 또는 이전 주문 라인에 적용됩니다.

다음 표에서는 각 라인에 나타날 수 있는 모든 필드에 대해 설명합니다. 그러나 사용 가능한 특정 필드는 현재 선택한 비용 영역에 따라 다릅니다.

| 필드 | 설명 |
|---|---|
| **통화** | 라인이 적용되는 통화를 선택합니다. 이 통화는 구매 주문과 관련이 있습니다. 시스템이 항해 및 항해 라인에 적용해야 하는 자동차 비용을 찾으려고 할 때 구매 주문과 동일한 통화를 가진 라인을 선택합니다. 이 필드는 **비용 영역** 필드가 *구매 주문* 또는 *항목* 으로 설정된 경우에만 사용할 수 있습니다. |
| **비용 유형 코드** | 비용 유형. |
| **배분 방식** | <p>비용을 라인에 분배하는 데 사용되는 방법입니다. 다음 옵션을 사용할 수 있습니다.</p><ul><li><p>**백분율** – **비용 값** 필드의 값은 상품의 총 가치에 적용되는 백분율입니다.</p><p>예를 들어 **비용 값** 필드가 *10* 으로 설정되어 있고 상품의 총 가치가 $800이면 비용 가치는 $80(= $800 × 10%)입니다.</p></li><li>**수량** - 비용은 상품 수량에 따라 배분됩니다.</li><li>**금액** - 비용은 상품 값에 따라 배분됩니다.</li><li>**볼륨** - 비용은 상품 볼륨에 따라 배분됩니다. 볼륨은 항목 마스터에 지정됩니다.</li><li>**무게** - 비용은 상품 무게에 따라 배분됩니다. 무게는 항목 마스터에 지정됩니다.</li><li>**체적** - 비용은 상품 체적에 따라 배분됩니다.</li><li><p>**측정** – 이 옵션을 사용하면 측정값을 **상륙 비용** 모듈에 지정할 수 있습니다. 개별 상품의 부피나 무게를 알지 못하지만 **금액** 및 **수량** 옵션에 허용된 것보다 정확한 배분이 필요한 조직에서 많이 사용합니다. 운송 주선인 또는 에이전트는 품목 수준 또는 구매 주문 수준에서 조직에 중량 또는 입방 측정값을 제공합니다.</p><p>예를 들어 해상 화물은 $900와 같습니다. 품목 A의 무게는 800킬로그램(kg)이고 부피는 2m³입니다. 품목 B의 무게는 100kg이고 부피는 1m³입니다. 비용을 가중치로 배분했을 때의 결과는 다음과 같습니다.</p><ul><li>항목 A = $800</li><li>항목 B = $100</li></ul><p>비용을 볼륨으로 배분했을 때의 결과는 다음과 같습니다.</p><ul><li>항목 A = $600</li><li>항목 B = $300</li></ul><p>**참고:** **배분 방식** 필드가 *측정* 으로 설정된 경우 시스템은 비용 영역(선적 컨테이너)과 라인 모두에 대해 입력된 측정값을 사용합니다. 이러한 측정값이 반드시 예상 총계에 합산될 필요는 없습니다. 그러나 예상 총계에 합산해야 하는 경우 통계를 사용하여 총 측정을 검토하여 확인할 수 있습니다. 선적 또는 컨테이너 수준에서 측정 프롬프트 설정 및 측정 자동 업데이트는 항해 헤더에서 설정됩니다.</p></li></ul> |
| **시작 날짜** | 일자 범위가 있는 경우 원가계산 일자 범위의 시작을 지정합니다. 이 날짜는 자동 비용이 적용되는 첫 번째 날짜입니다. |
| **종료 날짜** | 일자 범위가 있는 경우 원가계산 일자 범위의 끝을 지정합니다. 이 날짜는 자동 비용이 적용되는 마지막 날짜입니다. |
| **카테고리** | <p>비용을 계산하는 데 사용해야 하는 방법을 선택합니다. 다음 옵션을 사용할 수 있습니다.</p><ul><li>**고정** - 비용은 분배 방법에 따라 배분됩니다.</li><li>**조각** – 비용은 개당 할당됩니다. 따라서 할당 방법을 사용하지 않습니다.</li><li>**백분율** – 상품 가치의 일정 비율이 추가됩니다. 따라서 할당 방법을 사용하지 않습니다.</li><li>**비율** – 수량 내역이 있는 경우 이 옵션을 선택합니다. 라인의 **배분 방식** 필드는 *측정* 으로 설정해야 합니다.</li><ul> |
| **수량별로 구분** | <p>이 확인란을 선택하여 **수량 분석** 버튼을 **라인** 빠른 탭에서 사용할 수 있도록 합니다. 수량 할인을 사용하면 항해의 구매 주문서 라인에 있는 수량에 따라 비용을 분류하고 다른 비용을 변경할 수 있습니다. 이 기능은 배달 모드가 항공일 때 자주 사용됩니다. 라인의 **범주** 필드는 *비율* 로 설정해야 합니다.</p><p>수량은 **배분 방식** 필드에 선택된 옵션과 관련이 있습니다. 비용 값은 **비용 값** 필드에 입력된 수량까지입니다.<p>예를 들어, 45–100kg의 수량은 측정당 $6.00의 전하를 생성합니다(예: kg/m³). 100kg을 초과하는 수량은 측정당 $5.50의 전하를 생성합니다(예: kg/m³).</p> |
| **비용 가치** | 비용 값을 입력합니다. |
| **비용 통화 코드** | 비용의 통화를 선택합니다. |
| **품목 판매세 그룹** | 비용에 대한 세금 코드를 선택합니다. |
| **최소 비용** | <p>이 필드는 **수량별로 구분** 확인란이 선택된 경우에만 적용됩니다. 측정값이 이 값에 도달하지 않으면 **수량 분석** 페이지에 지정된 비용에 관계없이 최소값이 선택됩니다.<p>예를 들어, 0–45kg의 수량은 측정당 $6의 전하를 생성합니다(kg/m³). 46–100kg의 수량은 측정당 $5.50의 전하를 생성합니다(kg/m³). 2kg이 배송되는 경우 수량 분할은 $12의 비용 값을 생성합니다. 그러나 **최소 비용** 필드가 *$100* 으로 설정되면 최종 비용은 $100입니다.</p> |
| **집계** | 사용자가 이 비용을 배송 컨테이너 수준에서 항해 수준으로 이동할 수 있도록 하려면 이 확인란을 선택합니다. 이 경우 비용은 선적 컨테이너 수준에서 자동으로 계산될 수 있습니다. 그러나 개별 선적 컨테이너의 모든 품목 대신 항해의 모든 컨테이너에 있는 모든 품목에 대해 결합 및 배분할 수도 있습니다. |
| **연결 비용 유형** | <p>연결하려는 라인의 **비용 유형 코드** 값을 지정하여 동일한 자동 비용 레코드의 다른 라인에 현재 라인을 연결합니다. 이 기능은 현재 라인의 **범주** 필드가 *백분율* 로 설정된 경우에만 사용할 수 있습니다. 현재 회선이 다른 회선에 연결되어 있는 경우 현재 회선의 비용은 다른 회선의 비용을 기준으로 합니다.</p><p>예를 들어 운임은 $1,000이고 유류 할증료가 운임의 10%가 되기를 원합니다. 이 경우 라인에는 **범주** 값이 *백분율*, **비용 가치** 값이 *10%*, **연결 비용 유형** 값이 *화물* 이어야 합니다.</p> |
| **값 조정** 및 **조정 금액** | <p>이 필드를 사용하여 다양한 백분율 값의 값과 양을 조정합니다. **비용 영역** 필드가 *항목* 으로 설정된 경우에만 사용할 수 있습니다.</p><p>품목의 다른 구성요소에 대해 다른 원가계산을 설정할 수 있습니다. 항목의 한 구성 요소는 해당 항목의 다른 구성 요소와 비용 비율이 다를 수 있습니다. 이 접근 방식은 상품의 특정 부분을 서로 다른 요율로 평가하는 데 필요한 경우가 있습니다.</p><p>예를 들어, 시계의 관세는 두 가지 요율로 계산됩니다. 시계의 한 구성 요소에는 10%의 관세율이 적용되고 두 번째 구성 요소에는 2%의 관세율이 적용됩니다. 이 경우 원가 계산은 두 구성 요소로 분할됩니다.</p><p>비용은 항목에 대해 계산되지만 비용 범주가 다른 구성 요소의 값에 따라 비용 값이 조정됩니다. 나머지 구성 요소의 비용은 이전 계산이 조정된 금액을 사용하여 계산할 수 있습니다.</p><p>예를 들어, 시계의 구성 요소 A는 비용이 $9.50이고 관세가 10%이고 구성 요소 B는 비용이 $0.50이고 관세가 2%입니다. 따라서 총 비용은 $10.00입니다. 첫 번째 항목은 10% 라인에 대한 것입니다. 다음 값을 사용합니다.</p><ul><li>**범주:** *백분율*</li><li>**비용 가치** *10*</li><li>**조정된 값:** *조정 기준*</li><li>**조정된 값:** *-0.50*</li></ul><p>이 설정은 10% 관세가 계산되기 전에 품목의 비용($10)을 $0.50(구성 요소 B의 가격)만큼 줄여야 한다고 지정합니다. 따라서 $9.50에 10%가 적용됩니다.</p><p>두 번째 항목은 2퍼센트 라인(이전 항목이 조정된 $0.50)에 대한 것입니다. 다음 값을 사용합니다.</p><ul><li>**범주:** *백분율*</li><li>**비용 가치** *2*</li><li>**조정된 값:** *사용*</li><li>**조정:** *0.50*</li></ul><p>이 설정은 구성품 B에 대해 지불해야 하는 나머지 관세를 계산합니다.</p> |
